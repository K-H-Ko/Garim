# Garim Nginx 설정 가이드

통합 환경에서 프론트엔드와 백엔드를 이어주는 **Nginx 웹 서버 설정 파일**들이 모여있는 곳입니다.

## 📂 파일 구성

* **`nginx.conf`**: Nginx의 핵심 라우팅 및 서버 설정 파일입니다.
  * **도메인 연결**: `garim.shop` 및 `www.garim.shop` 요청을 수신합니다.
  * **프론트엔드 (`/`)**: Vite 프론트엔드 컨테이너(`final_frontend:3000`)로 요청을 전달하며, 원활한 실시간 통신을 위해 WebSocket 기능을 지원합니다.
  * **백엔드 (`/api/`)**: FastAPI 백엔드 서버(`final_backend:8000`)로 API 요청을 라우팅합니다.
  * **대용량 처리**: 영상 파일 업로드를 위해 용량 제한을 무제한(`client_max_body_size 0;`)으로 해제해 두었습니다.
* **`Dockerfile`**: 커스텀 Nginx 도커 이미지를 빌드하기 위한 설정 파일로, 우리가 작성한 `nginx.conf`를 Nginx 서버 내부에 덮어씌워 줍니다.

## ⚙️ 역할 및 특징

- **단일 진입점(Reverse Proxy)**: 클라이언트(스마트폰 브라우저, 외부 코랩 워커 등)는 포트 번호를 복잡하게 알 필요 없이 **정식 도메인(`https://garim.shop`)** 또는 로컬(`http://localhost`) 주소 하나로만 접속하면 됩니다. Nginx가 요청 경로(`/`는 프론트엔드로, `/api/`는 백엔드로)를 파악해 내부에서 알아서 트래픽을 분배하는 교통경찰 역할을 합니다.
- **Cloudflare Zero Trust 연동**: 외부망에서 들어오는 `https://garim.shop` 트래픽은 Cloudflare 터널을 거쳐 최종적으로 이 Nginx로 안전하게 전달됩니다. 덕분에 복잡한 포트포워딩(공유기 설정) 없이도 완벽한 HTTPS 보안 환경에서 전 세계 어디서든 서비스가 가능합니다.
- **대용량 파일 업로드 최적화**: 고화질 영상 파일이 Nginx 단에서 튕기는 현상을 막기 위해 파일 업로드 크기 제한을 완전히 해제하여 무거운 데이터도 안정적으로 전송됩니다.
