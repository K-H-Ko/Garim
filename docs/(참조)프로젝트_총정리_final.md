# GARIM 프로젝트 비즈니스 기획 및 기술 아키텍처 총정리 (Final)

## 1. 프로젝트 발전 과정 요약 (v1 → v2 → v3 → Final)

GARIM 프로젝트는 개인정보 유출로 인한 '퍼즐 도싱(Doxxing)'을 방지하기 위해 시작되었습니다. 다음과 같은 과정을 거쳐 현재의 최종 형태로 완성되었습니다.

- **v1 (초기 기획)**: 영상미를 훼손하지 않는 인페인팅 마스킹과 오디오 Beep 마스킹의 기본 개념 확립.
- **v2 (구조 개편)**: 무거운 AI 모델 처리를 위해 **단일 파이프라인에서 5분할 파이프라인**으로 재설계. `result.json`을 마스터 데이터로 삼아 프론트와 백엔드 간의 병목 현상 해소.
- **v3 (흐름 최적화)**: 사용자 경험 개선을 위해 7페이지 구조를 6페이지로 압축(Processing과 Download 통합). 분석은 무료로 제공하되, 상세 확인 시 크레딧을 차감하는 비즈니스 모델(BM) 확립.
- **Final (현재)**: 로컬 백엔드의 자동화 워커(`local_worker.py`)와 Colab GPU 워커가 유기적으로 연동되어 사용자 개입 없이 완전 자동화된 E2E(End-to-End) 파이프라인 구축 완료.

## 2. 서비스 기획 및 비즈니스 목적

### 2.1 기획 배경
개인정보 유출은 DB 해킹뿐 아니라, SNS나 브이로그에 무심코 노출된 **택배 송장, 번호판, 통화 소리** 등 파편화된 정보가 모여 신상이 특정되는 **'퍼즐 도싱'**으로 발생합니다. 기존의 일괄 모자이크 방식은 영상미를 크게 훼손하여 사용자들이 보안 처리를 기피하게 만들었습니다.

### 2.2 핵심 가치 및 타겟
- **인페인팅 마스킹**: 텍스트 영역을 주변 픽셀로 자연스럽게 채워 이질감을 없앰.
- **오디오 단어 단위 마스킹**: 화자의 대화를 끊지 않고, 유출되는 핵심 단어 구간에만 1000Hz Beep 음을 합성.
- **타겟 고객**: 영상미를 중시하는 **인플루언서 및 크리에이터** (구독 및 크레딧 결제 고객), 그리고 일상 영상을 안전하게 공유하려는 일반 대중.

## 3. 최종 처리 아키텍처 (5분할 파이프라인)

무거운 AI 처리를 분산하기 위해 로컬 CPU와 Colab GPU가 역할을 나누어 수행합니다.

| 모듈명 | 실행 환경 | 핵심 역할 | 주요 산출물 |
|--------|----------|----------|------------|
| `OCR_pipeline_report.py` | 로컬 (CPU) | 영상/이미지 내 텍스트 및 시각 PII 탐지 | `_index.json` |
| `colab_pipeline_stt.py` | Colab | 오디오 추출 후 음성 인식 및 음성 PII 탐지 | `_stt.json` |
| `backend_json_merger.py` | 로컬 | 시각+음성 결과 병합, 재생바 마커 생성 | **`result.json` (핵심)** |
| `pipeline_detail_view.py` | 로컬 (CPU) | 프론트엔드용 상세보기 영상 및 오버레이 선(先)생성 | `_상세보기.mp4`, `_tracks.json` |
| `colab_pipeline_mask.py` | Colab (GPU) | 사용자가 선택한 PII 영역만 최종 인페인팅 및 Beep 마스킹 | 최종 마스킹 영상/이미지 |

## 4. 최종 사용자 이용 흐름 (6단계 UI)

사용자 관점에서 서비스는 다음 6단계의 직관적인 흐름으로 동작합니다.

| 단계 | 화면 (페이지) | 사용자 동작 | 시스템(백엔드) 처리 내용 | 크레딧 결제 |
|---|---|---|---|---|
| **1** | **Upload** | 파일 선택 및 "분석 시작" | 청크 업로드 후 `analysis` Job 큐 등록 | 무료 (0 크레딧) |
| **2** | **AnalysisProgress** | 진행률 대기 | 로컬 워커가 OCR, 병합, PII 등록 자동 수행 | - |
| **3** | **AnalysisReport** | 위험도 요약 확인 | 전체 탐지 건수 및 위험도만 노출 (상세 블라인드) | - |
| | | "상세보기" 클릭 | **[결제 로직]** 사용자 크레딧 차감 | **이미지 2 / 영상 3 차감** |
| **4** | **ReplaceOptions** | 가릴 항목 체크박스 선택 | 이미지: Canvas 오버레이 / 영상: tracks.json 렌더링 | - |
| **5** | **Preview** | 마스킹 전/후 비교 | 가림바(ComparisonSlider)로 좌우 비교 (영상은 6초 클립) | - |
| **6** | **ResultPage** | 결과 확인 및 다운로드 | `mask_final` Job 실행 후 최종 파일 다운로드 제공 | - |

## 5. 핵심 기술 및 인프라 요약

- **프론트엔드**: React, Vite, TailwindCSS (Canvas 기반의 동적 Bbox 오버레이 및 영상 동기화 UI 구현)
- **백엔드**: FastAPI, PostgreSQL, Redis (비동기 Job Queue 관리 및 청크 파일 업로드 처리)
- **자동화 워커**: `backend/local_worker/local_worker.py`가 백엔드와 생명주기를 같이하여 5초 단위로 Job을 폴링, 외부 개입 없이 분석 파이프라인 자동 구동.
- **인증 및 결제**: OAuth 기반 JWT 토큰 (Redis 세션 관리). Toss Payments 연동을 통한 크레딧/구독 분리 결제 아키텍처.
- **배포 인프라**: Docker Compose를 활용하여 Nginx 리버스 프록시 뒤에서 프론트와 백엔드 컨테이너화 구동.
