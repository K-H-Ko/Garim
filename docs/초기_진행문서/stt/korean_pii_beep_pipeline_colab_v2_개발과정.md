# korean_pii_beep_pipeline_colab_v2 개발 과정 및 시행착오 정리

## 프로젝트 목표

한국어 음성에서 개인정보를 탐지하고 해당 구간만 정확하게 삐(Beep) 처리하는 파이프라인을 구축한다.

최종 목표는 다음과 같다.

```text
영상 입력
↓
음성 추출
↓
STT 수행
↓
개인정보 탐지
↓
개인정보 구간만 Beep 처리
↓
원본 영상과 합성
↓
결과 영상 생성
```

---

# 1차 시도 - STT 텍스트 기반 개인정보 탐지

## 접근 방식

처음에는 STT 결과 텍스트만을 기반으로 개인정보를 탐지하였다.

예시:

```text
유재석입니다.
전라남도 목포시 통일대로 140에 삽니다.
010-2345-2222입니다.
```

이 텍스트에서

- 이름
- 주소
- 전화번호

를 찾아 개인정보로 판단하였다.

---

## 문제점

### 전화번호

비교적 잘 탐지됨

```text
010-2345-2222
```

형태는 Regex로 쉽게 검출 가능

---

### 주소

정확도가 낮음

예시

```text
전라남도 목포시 통일대로 140
```

가

```text
전라남도 목포시 통일 대로 백사십
```

또는

```text
목포시 통일대로
```

처럼 인식되는 경우가 많았음

주소는 STT 오인식 영향을 크게 받음

---

### 이름

문맥이 없으면 탐지가 어려움

예시

```text
유재석입니다.
```

는 비교적 쉬움

하지만

```text
유재석 씨가
```

```text
유재석은
```

```text
유재석 기자는
```

등 다양한 표현이 존재

단순 Regex만으로는 한계가 있었음

---

# 2차 시도 - Segment 단위 Beep 처리

## 접근 방식

STT Segment Timestamp 사용

예시

```python
segment.start
segment.end
```

를 이용하여 개인정보가 포함된 Segment 전체를 Beep 처리

---

## 문제점

과도한 Beep 처리 발생

예시

```text
안녕하세요.
제 번호는 010-2345-2222입니다.
감사합니다.
```

이 하나의 Segment로 묶이면

```text
안녕하세요.
제 번호는 010-2345-2222입니다.
감사합니다.
```

전체가 Beep 처리됨

원하는 결과는

```text
안녕하세요.
제 번호는
[삐]
입니다.
감사합니다.
```

# 3차 시도 - Word Timestamp 활용

## 개선 목표

개인정보 부분만 정확하게 Beep 처리

---

## 적용 내용

Whisper의

```python
word_timestamps=True
```

옵션 사용

예시

```python
[
    ("제", 0.10, 0.20),
    ("번호는", 0.20, 0.50),
    ("010", 0.50, 0.80),
    ("2345", 0.80, 1.10),
    ("2222", 1.10, 1.40)
]
```

형태로 단어별 시간 정보를 확보

---

## 결과

전화번호 시작 시점과 종료 시점을 정확하게 계산 가능

```text
010-2345-2222
```

부분만 Beep 처리 가능해짐

# 7차 시도 - 최종 Beep 파이프라인 구성

## 최종 구조

```text
영상 입력
↓
오디오 추출
↓
Whisper STT
↓
Word Timestamp 생성
↓
Regex 탐지
↓
NER 탐지
↓
개인정보 구간 계산
↓
Beep 생성
↓
오디오 합성
↓
영상 재합성
↓
최종 영상 출력
```

# 최종 버전(v2)의 특징

Segment 단위가 아닌

```text
단어 단위 Timestamp
```

활용

---

## 2. Regex + NER 결합

전화번호

```text
Regex
```

이름/주소

```text
NER + Regex
```

조합 사용

---

## 3. 필요한 구간만 Beep 처리

기존

```text

```

으로 개선

## 4. 영상 재합성 지원

```text

+

```

를 결합하여 최종 결과 영상 생성
