# Garim 프로젝트 결정사항 메모

이 파일은 대화에서 정한 주요 결정을 잊지 않기 위해 최신화하는 작업 메모입니다.

## DB 설계 운영 정책

- DB에는 실제 업로드/결과 파일 바이너리를 저장하지 않고, 파일 경로와 해시, 크기, MIME 타입 같은 메타데이터만 저장한다.
- 진행률은 매번 갱신하지 않고 `analysis_jobs.progress_percent`를 5% 단위 변화가 있을 때만 업데이트한다.
- 대시보드 통계는 실시간으로 계속 조회하지 않고 기본 10초에 한 번씩 조회한다.
- 반복 집계가 필요한 값은 캐시 또는 요약 데이터를 사용하고, 원본 테이블 전체 count를 매번 돌리지 않는다.
- 자동 삭제는 PPT 정책을 기준으로 진행한다.

## 보관 정책

- 업로드 원본 파일: 처리 완료 후 12시간 보관
- Free 결과 파일: 7일 보관
- 1회권 결과 파일: 30일 보관
- 구독/플랜 결과 파일: 90일 보관
- 메타데이터/분석 요약: 워터마크 역추적 및 감사 목적 90일 보관
- 회원 탈퇴: 7일 유예 후 개인정보 삭제 또는 익명화
- 삭제 완료/실패 이력은 `deletion_events`에 기록한다.

## SNS 범위

- 현재 구현 범위에서는 Instagram OAuth를 제외한다.
- 추후 확장 가능성을 위해 `sns_connections`, `sns_media_items`, `sns_diagnosis_jobs` 테이블은 DB 설계에 포함한다.
- Kakao, Google, Facebook, X OAuth는 실제 로그인 연동 대상으로 유지한다.
