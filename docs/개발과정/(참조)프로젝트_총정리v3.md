## 1. 서비스 기획 배경 및 비즈니스 목적 (유지)

### 1.1 기획 배경: '퍼즐 도싱(Doxxing)' 차단
개인정보 유출은 DB 해킹뿐 아니라, SNS/브이로그에 무심코 노출된 영상 속 **택배 송장·번호판·
주소판·통화 소리** 같은 파편 정보가 모여 신상·거주지가 특정되는 **'퍼즐 도싱'**으로 발생한다.
기존 일괄 모자이크는 영상미를 훼손해 사용자가 보안 처리를 기피하게 만들었다.

### 1.2 타겟 고객
- **메인(인플루언서·크리에이터)**: 영상미를 100% 보존하며 개인정보만 자연스럽게 지우는 솔루션에
  비용(구독·크레딧)을 지불할 고가치 고객.
- **서브(일반 대중)**: 중고거래 인증·일상 SNS 업로드 시 유출 예방. 무료 진단 리포트로 유입.

## 2. 기술 요구사항 (유지)

1. **YOLOv11·SAM2·STE 모델 배제** — GPU 부하/지연 방지로 미사용 확정.
2. **인페인팅 마스킹** — 텍스트 박스 영역을 주변 픽셀로 자연스럽게 채워 이질감 제거. (블러/스티커 미지원, 추후 예정)
3. **단어 단위 Beep 오디오 마스킹** — Whisper `word_timestamps` + NER/정규식으로 핵심 단어의
   ms 단위 구간에만 1000Hz Sine Beep 오버레이(대화 단절 없음).

## 3. ★ 처리 아키텍처: 5분할 + result.json 중심 (v2 유지)

### 3.1 모듈 구성

| 모듈 | 환경 | 역할 | 산출물 |
|------|------|------|--------|
| `OCR_pipeline_report.py` | **로컬 CPU** | 시각 PII(OCR) 탐지 | `{stem}_index.json` |
| `colab_pipeline_stt.py` | **Colab** | 음성 STT+PII 탐지 | `{stem}_stt.json` |
| `backend_json_merger.py` | 로컬 | 시각+음성 병합, 마커 생성, 상세보기 자동 실행 | `{stem}_result.json` |
| `pipeline_detail_view.py` | 로컬 CPU | 상세보기 선(先)생성 | `_상세보기.mp4`, `_tracks.json` |
| `colab_pipeline_mask.py` | **Colab GPU** | 선택분만 마스킹 | 최종 마스킹본 |

### 3.2 ★ result.json = 단일 마스터 데이터

모든 후속 단계(리포트·상세보기·선택·마스킹)가 이 한 파일을 기준으로 동작한다.
- `pii_groups[]` (시각): 좌표(bbox/polygons/keyframes), 위험도, `is_selected`, `pii_id`
- `audio_pii_groups[]` (음성): 발화 구간(start/end_time_sec), detected_text
- `timeline_markers[]`: 재생바 마커
- **`is_selected`** (초기 false) → 사용자가 켠 항목만 마스킹

### 3.3 자동화 워커 — local_worker.py (★ v3 신규)

`python ocr_mask/local_worker.py` 상시 실행 시 5초 폴링으로 Job을 자동 감지·처리.
- `analysis` Job: OCR → 병합 → PII/artifact 등록 → complete
- `mask_preview/mask_final` Job: 더미 처리(원본 경로 등록) → complete *(추후 Colab GPU로 교체 예정)*

## 5. ★ 페이지별 상세 동작 흐름

### ① Upload 페이지

**[목적]**: 파일을 안전하게 청크 단위로 업로드하고 분석 Job을 큐에 등록한다.

```
사용자: 파일 선택(드래그/클릭)
  → 프론트 유효성 검사 (확장자: mp4/avi/mov/mkv/jpg/png/jpeg/webp)
  → POST /uploads/init        (세션 생성, upload_id 반환)
  → POST /uploads/{id}/chunks/{i} × N (5MB 청크 반복 전송)
  → POST /uploads/{id}/complete (병합)
  → POST /analysis/jobs        (Job 생성, status=queued, 크레딧 0)
  → 이동: /analysis-progress?jobId=...
```

### ② AnalysisProgress 페이지

**[목적]**: 백그라운드 분석 파이프라인 진행률을 실시간으로 표시하며 완료를 대기한다.

```
프론트: GET /analysis/jobs/{id} 2.5초 폴링
  → status: queued → processing → completed 단계 표시

백엔드 (local_worker.py 자동 처리):
  1. GET /worker/jobs/next       (5초 폴링)
  2. POST /worker/jobs/{id}/accept
  3. OCR_pipeline_report.run_pipeline() → index.json
  4. backend_json_merger.merge() → result.json (+ pipeline_detail_view 자동 실행)
  5. POST /worker/jobs/{id}/results/pii      (visual_pii 등록)
  6. POST /worker/jobs/{id}/results/artifact (result.json, 상세보기.mp4, tracks.json)
  7. POST /worker/jobs/{id}/complete

완료 시 → /analysis-report?jobId=...
```

### ③ AnalysisReport 페이지 (★ 전면 재설계 예정)

**[목적]**: 무료 사용자도 접근 가능한 탐지 요약 리포트. 상세 내용은 크레딧 차감 후 공개.

```
프론트: GET /analysis/jobs/{id}/detections
  → 건수(N건) + 위험도(High/Medium/Low) 게이지만 표시
  → 상세 카테고리/타임라인 비공개

"상세보기" 버튼 클릭
  → CreditConfirmModal 팝업 (이미지 2크레딧 / 영상 3크레딧 안내)
  → 확인: POST /analysis/jobs/{id}/detail-access
     → 크레딧 있음: /replace-options?jobId=... 이동
     → 크레딧 없음: /subscription 이동
```

### ④ ReplaceOptions 페이지 (★ 이미지/영상 완전 분기 예정)

**[목적]**: 탐지된 PII를 시각적으로 확인하고 마스킹할 항목을 직접 선택한다.

```
공통: GET /analysis/jobs/{id}/detections + GET /analysis/jobs/{id}/result

[이미지 분기]
  → 원본 이미지 + Canvas 오버레이
  → 각 PII bbox 박스 표시 (라벨: 주소1, 주민번호1 …)
  → 체크박스로 마스킹 항목 선택
  → "미리보기 생성" 버튼 → PUT /selections → /preview 이동

[영상 분기]
  → 원본 영상 인라인 재생 + Canvas 오버레이
  → tracks.json 로드 → timeupdate 이벤트로 bbox 실시간 렌더링
  → 재생바 위 PII 탐지 시점 마커
  → PII별 체크박스 + 개별 "미리보기" 버튼 (→ /preview 이동, 해당 pii_id + 6초 구간 전달)
  → "처리진행" 버튼 → PUT /selections → /result 이동
```

### ⑤ Preview 페이지 (★ 가림바 재설계 예정)

**[목적]**: 마스킹 적용 전/후를 한 화면에서 좌우 비교한다. 가림바를 드래그해 비율 조정.

```
[이미지 미리보기]
  → POST /analysis/jobs/{id}/mask-preview → Job 폴링
  → 완료: 원본/마스킹 이미지 가림바 비교 (clip-path: inset)
  → 버튼: 뒤로가기 / 처리진행(→ /result)

[영상 6초 클립 미리보기]
  → 상세 페이지에서 pii_id + clip_start/clip_end state 전달
  → POST /analysis/jobs/{id}/mask-preview (pii_id, clip_start, clip_end)
  → 완료: 두 <video> 겹침 + clip-path 분할 + currentTime 동기화
  → 버튼: 뒤로가기만 (처리진행은 상세 페이지에)
```

### ⑥ ResultPage 페이지 (★ 신규 생성 예정 — Processing+Download 통합)

**[목적]**: 마스킹 처리가 완료될 때까지 대기하고, 완료 시 최종 결과물 미리보기와 다운로드를 제공한다.

```
[이미지]
  → mask_final Job 생성 (이미지는 빠름)
  → 즉시 결과 표시 + 다운로드 버튼

[영상]
  → POST /analysis/jobs/{id}/mask-final
  → Job 폴링 (processing → completed)
  → 완료: 최종 마스킹 영상 인라인 미리보기 (워터마크 없음)
  → GET /analysis/jobs/{id}/download (FileResponse)
  → 만료일 표시 (plans.result_retention_days 기준)
  → "다른 파일 처리" 버튼 → /upload
```

## 7. 결과물 제공 방식 (v3 업데이트)

- **상세보기**: merge 직후 만들어둔 `_상세보기.mp4`+`_tracks.json`을 즉시 표시 → 대기 없음.
- **미리보기**: 원본/마스킹 결과를 **가림바(ComparisonSlider)** 로 좌우 비교. clip-path로 분할 비율 드래그 조정.
  - 이미지: 전체 1장 마스킹 미리보기
  - 영상: 선택한 PII 구간 6초 클립만 마스킹 미리보기 (빠른 응답)
- **본처리 (ResultPage)**: `is_selected=true` 항목만 전체 마스킹. 워터마크 없는 최종 파일 제공.
  - ~~미리보기=보이는 워터마크, 최종=안 보이는 워터마크~~ (현재 미적용, 추후 검토)
- **다운로드**: 로그인 필수, 본인 Job만 접근 가능. 보관기간 내(`expires_at`) 횟수 제한 없이 다운로드.

## 10. ★★★ 개발예정 (Phase 2 → Phase 3 순서)

> 아래 항목은 미구현 상태. `(참조)프론트_백엔드연결.md`의 "진행예정" 섹션에 파일별 상세 명시됨.
