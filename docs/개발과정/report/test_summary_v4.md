# 🏆 개인정보(PII) 비식별화 및 배경 복원 파이프라인 개발 종합 요약서 (v4)

이 문서는 프로젝트 초기 기획 및 요구사항 정의 단계부터, AnyText 시도 및 실패, 객체 탐지, 폰트 합성 시도 및 전면 폐기, 그리고 **Stable Diffusion(디퓨전) 모델 도입과 3대 핵심 측정법을 결합한 하이브리드 배경 복원**까지의 전체 진화 과정을 흐름 순서로 일목요연하게 정리한 종합 보고서입니다.

---

## 1. 📋 프로젝트 개요 및 목표

- **서비스 목적:** 개인정보 보호 자동화 서비스 구축.
- **최종 목표:** 신분증, 우편물, 택배송장 등 다양한 문서 이미지 내 개인정보(PII)를 정확하게 탐지하고, 주변 원본 텍스트 및 배경의 어떠한 훼손도 없이 감쪽같이 지우고 대체(배경 복원)하는 AI 자동화 파이프라인 구축.
- **최우선 타겟:** 실물 촬영 시 훼손 위험이 가장 높은 **'영수증', '택배송장', '신분증'** 3대 문서 이미지.
- **실행 환경:** Google Colab (GPU 사용), 모델 가중치는 Google Drive 마운트 경로(`/content/drive/MyDrive/final_PJ_model/`) 활용.

## 2. 🚀 전체 개발 진화 흐름 및 단계별 구현 내역

### 📍 [Phase 0] AnyText STE 최초 시도 및 실패 → 파이프라인 전환
- **시도:** AnyText(Scene Text Editor) 모델 하나로 OCR·배경 지우기·가짜 텍스트 합성을 일괄 처리 시도.
- **실패:** `expected scalar type Float but found Half` dtype 불일치 및 한국어 지원 한계(중국어/영어 위주).
- **결론:** AnyText 의존도를 낮추고, OCR·배경 복원 컴포넌트를 분리하는 하이브리드 파이프라인으로 전환.

### 📍 [Phase 1] OCR 엔진 선택 및 고도화
- **채택:** 한국어 정확도가 가장 높은 **Naver Clova OCR**을 메인으로, PaddleOCR을 폴백으로 채택.
- **캐싱:** 반복 API 호출에 따른 비용/시간 낭비를 막기 위해 좌표 결과를 JSON 파일로 캐싱 시스템 도입.

### 📍 [Phase 2] YOLO + SAM2 객체 탐지 파이프라인
- **탐지:** YOLO 모델로 개인정보 존재 가능 영역(ROI) 1차 탐지 후, SAM2로 픽셀 단위 정밀 마스킹 진행. 탐지 실패 시 원본 전체 OCR 진행.

### 📍 [Phase 3] 텍스트 정밀 추출 및 PII 탐지 엔진
- **정규식 엔진:** 주민등록번호, 주소, 카드번호 등 핵심 개인정보 패턴 완벽 탐지.
- **2차 정제 필터:** 주소 중간 띄어쓰기나 숫자 혼합 시 탐욕적(Greedy)으로 쓸어 담은 후 정밀하게 걸러내는 2차 필터 구축.

### 📍 [Phase 4] 맥락 보존형 가짜 데이터(Fake Data) 생성 엔진
- **의미적 치환:** 단순 정규식 1:1 치환을 넘어, Gemini LLM을 연동하여 실제 존재하는 듯한 자연스러운 한국형 주소와 명칭 생성.

### 📍 [Phase 5] 초정밀 픽셀 타겟팅 기반 배경 복원 (LaMa 인페인팅)
- **도입:** 단순 사각형 덮어쓰기 시 배경 무늬 훼손 방지를 위해 LaMa 모델 최초 채택.
- **이진화:** Otsu 이진화로 검은색 잉크 픽셀만 정밀 마스크로 추출 후 LaMa로 텍스트 부분만 복원 시도.

### 📍 [Phase 6] 폰트 매칭 및 3D 합성 시도 (6/1부로 전면 폐기)
- **시도:** 150종 폰트의 밀도와 장평을 수학적으로 매칭하고 `cv2.warpPerspective`로 3D 비틀기 합성을 시도.
- **결과 (폐기 결정):** 정교한 모델 없이는 폰트 매칭의 이질감이 커서 티가 많이 남을 인지. **결국 오늘(6/1) 가짜 텍스트 '합성' 관련 코드는 전면 폐기하고, 오직 "완벽한 제거 및 배경 복원"에만 집중**하기로 방향 대폭 수정.

### 📍 [Phase 7] 물리적 고도화 패키지 도입
- **격자선 보호:** 모폴로지 기법으로 표의 가로/세로 격자선 및 경계 그림자를 100% 보존.
- **2중 격리 보호막:** 지우지 말아야 할 주변 원본 텍스트에 사방 3px 마진을 더해 훼손 0% 달성.

## 3. 🛡️ [핵심 보완] 디퓨전(SD) 도입 및 하이브리드 파이프라인 구축 (Phase 8 ~ 9 최신)

오늘(6/1) 합성 기능을 폐기한 직후, 가장 감쪽같은 배경 복원을 달성하기 위해 **Stable Diffusion 인페인팅 모델 도입**과 **다중 모델 결합**이라는 거대한 아키텍처 개편이 이루어졌습니다.

### 📍 [Phase 8] 디퓨전 모델 도입 및 3대 측정법 기반 하이브리드 분기
- **디퓨전(Stable Diffusion) 로컬 연동:** 코랩 로딩 시간 단축을 위해 SD 인페인팅 모델을 로컬 드라이브에 직접 다운로드하여 구축.
- **문제 발생:** SD 모델 단독 사용 시, 지정된 텍스트 범위 외의 이미지 전체가 훼손되거나 형태가 틀어지는 문제 발견.
- **3대 핵심 측정법 도입:** 배경을 정확히 구분하여 최적의 복원 모델을 투입하기 위해, ① **엣지 밀도 측정**, ② **픽셀 대비 및 분산 측정**, ③ **형태학적 구조선 감지** 기법을 적용한 하이브리드 자동 분기 시스템 설계.
- **하이브리드 교차 결합:** 디퓨저(SD)가 가장 잘하는 영역, LaMa가 잘하는 영역, Seamless Clone이 잘하는 영역을 혼합하여 **SD + LaMa + Seamless**를 교묘하게 섞어 쓰는 최고의 배경 복원 파이프라인 가동 (이 과정에서 발생한 각종 에러 트러블슈팅 완료).
- Stable Diffusion 모델의 배경 생성시 텍스트를 임의로 생성하는 단점이 발생하여 **LaMa + Seamless** 만 사용하는 것으로 결정.

### 📍 [Phase 9] 정밀 마스킹 튜닝
- **정밀 타겟팅 및 훼손 방지:** 주변 텍스트 훼손 없이 바운딩 박스 안에서만 정교하게 지우도록 마스킹 범위를 튜닝하여 인접 글자 보호 성공.

## 4. ⚠️ 현재 한계점 및 향후 계획

### 1. 일부 텍스트 잔상 문제 (처리 방법 치열하게 고민 중)
- **현재 이슈:** 여러 마스킹 기법 도입과 디퓨저/LaMa 결합 노력에도 불구하고, **아직도 텍스트 일부가 깨끗하게 지워지지 않고 미세한 잔상(얼룩, 잉크 찌꺼기 등)이 남는 문제**가 존재합니다.
- **해결 과제:** 안티앨리어싱이나 엠보싱 주변 노이즈를 1픽셀 오차 없이 완벽히 지워버리기 위한 마스킹 방식 변경 및 임계값 튜닝 방법을 찾아야함.

### 2. 향후 계획 (영상 비식별화로 확장)
- **영상 단위 처리:** 단일 이미지 기반 PII 마스킹 파이프라인 품질이 안정화되면, **차후에는 동영상 단위에서 프레임별로 개인정보를 찾아 지금처럼 마스킹하는 영상 비식별화 시스템**으로 확대 적용할 예정.
