## 2. 발견된 문제점 (3가지)

### 문제 ① — 보간(Interpolation) 완전 생략 [가장 심각]

**현상**: 카드가 화면에 존재하는 수백 프레임 중 OCR 키프레임(29프레임) 구간만 마스킹되고,  
나머지 ~320프레임은 카드번호가 그대로 노출됨.

**근본 원인**: `_VID_MAX_INTERP_GAP = 2.0초` 고정 임계값 문제  
- OCR JSON은 약 90프레임(3초)마다 1개씩 생성 → 모든 keyframe 간격이 3~6초
- 2.0초 임계값을 초과하면 보간을 건너뛰는 로직 → **6쌍 중 4쌍 보간 생략**
- 결과: 전체 513프레임 중 약 212프레임만 마스킹 (나머지 카드번호 노출)

**해결책**: 고정 시간 임계값 완전 제거 → **OCR 부재 증거 기반 판단 로직**으로 교체
- 두 keyframe 사이에 "PII가 감지되지 않은 OCR 프레임"이 2개 이상 연속 → 카드 이탈 확정 → 첫 부재 프레임 직전까지만 마스킹
- 그 외 모든 경우: 보간 수행 (보안 우선 과잉 마스킹 원칙)

---

### 문제 ② — 유령 잔상(Ghost Patch) 아티팩트

**현상**: 카드가 없는 어두운 장면에서 하얀/회색 반투명 얼룩 덩어리가 화면에 떠다님.

**근본 원인**: 기존 LaMa 캐시 방식의 설계 결함  
- keyframe A의 배경을 LaMa로 복원 후 캐시에 저장
- 이후 카드가 다른 위치로 이동한 프레임에도 동일한 캐시 배경을 붙여넣음
- 위치 불일치 → "얼어붙은 배경 패치"가 잘못된 위치에 표시되어 유령처럼 보임

**해결책**: LaMa 캐시 방식 폐기 → **cv2.inpaint (Telea 알고리즘)**으로 완전 교체  
- 각 프레임의 주변 픽셀을 실시간으로 분석하여 마스크 영역을 채움
- 캐시 없음 → 프레임마다 독립 처리 → 유령 패치 구조적 불가능

---

### 문제 ③ — 잔상 지속 (카드 사라진 후에도 마스킹 유지)

**현상**: 카드가 이미 화면에서 사라진 장면에서도 마스킹 패치가 계속 남아 있음.

**근본 원인**: 마지막 keyframe 이후 `max_f = fps × 1.0초` 고정으로 마스킹 연장  
- 카드가 실제로 언제 화면에서 나갔는지와 무관하게 무조건 1초 더 마스킹

**해결책**: 마지막 keyframe 이후 구간도 **OCR 부재 증거 기반 종료** 적용  
- 마지막 keyframe 이후 첫 번째 "PII 미탐지 OCR 프레임" 직전에서 마스킹 종료
- 카드_영상1 기준: 마지막 키프레임 480 → 부재 OCR 510 → 프레임 509에서 정확히 종료

### CSRT 추적 원리

```
각 keyframe에서 CSRT 추적기 초기화
  ↓
다음 프레임: CSRT.update(frame) 실행
  ├─ 성공 → 추적된 새 위치로 개별 박스 이동 → polygon 마스킹
  └─ 실패 → 카드 이탈/장면 전환 확정 → 즉시 마스킹 중단
```

카드 텍스트 박스(1234, 5678, 9012, 01/29 등)를 각각 독립 polygon으로 마스킹.  
CSRT가 합산 bbox 이동량을 추적 → 각 박스를 동일한 오프셋(dx, dy)만큼 이동.

## 10. 4차 코드 수정 — ORB 아핀 변환 도입 (회전 잔상 문제 해결)

### 문제 (0:00:13 잔상 보임)

사용자 보고: "지워졌으면 계속 그 상태여야 하는데 잔상이 잠시 보인다"

**근본 원인 분석:**

| 현상 | 원인 |
|------|------|
| 카드가 회전할 때 텍스트 박스가 실제 위치에서 벗어남 | `_shift_boxes(dx, dy)` = 이동만 처리, 회전 변환 없음 |
| 회전 각도만큼 박스가 실제 텍스트보다 떨어진 위치에 마스킹됨 | 카드 중심 기준 오프셋이 회전하면 (dx, dy)만 이동으론 보정 불가 |
| 결과: 실제 카드 텍스트가 마스크 바깥에 노출 → 잔상 | CSRT 추적 성공임에도 박스 위치가 틀려서 발생 |

**예시 (카드 30° 회전 시 오차):**
- 텍스트 박스가 카드 중심에서 50px 떨어져 있을 때
- 순수 이동 shift 오차 = 약 25px → pad=3으로 커버 불가

### 수정 내용

**추가한 함수**: `_transform_boxes_affine(boxes, M)`
- 아핀 변환 행렬 M을 각 텍스트 박스의 꼭짓점(vertices)에 직접 곱셈 적용
- 회전 + 스케일 + 이동을 모두 처리하는 `estimateAffinePartial2D` 기반

**키프레임 초기화 시 추가**:
- ORB 특징점 추출기 초기화 (`cv2.ORB_create(500)`)
- 카드 ROI에서 ORB 키포인트 + 디스크립터 추출 후 state에 저장

**추적 프레임 업데이트 로직 교체**:
```
이전: _shifted = _shift_boxes(orig_boxes, dx, dy)  ← 이동만

이후:
  ① CSRT가 현재 프레임 카드 위치 추적 → bbox 획득
  ② 해당 bbox 영역에서 ORB 특징점 추출
  ③ 키프레임 ORB ↔ 현재 프레임 ORB 매칭 (BF Matcher)
  ④ 4개 이상 매칭 → estimateAffinePartial2D로 M 계산 (RANSAC)
  ⑤ M을 orig_boxes 각 꼭짓점에 적용 → 회전된 정확한 위치
  ⑥ 매칭 실패 시 → _shift_boxes fallback
```

### 변경 파일 및 위치

  - `_transform_boxes_affine()` 함수 추가 (라인 ~843)

## 11. 5차 코드 수정 — "한번 지우면 유지" 원칙 적용

### 문제 (0:00:08 잔상)

**근본 원인**: CSRT가 5프레임 연속 실패하면 `state['active'] = False`로 마스킹을 **완전히 중단**하는 로직

```
키프레임 90(3s) → CSRT 추적 → 장면 전환(~5s경) → CSRT 연속 5회 실패
→ active=False → 마스킹 중단 → 0:00:08 카드 잔상 노출
```

### 수정 내용

**CSRT 실패 처리 3단계 구조:**

| fail_count | 이전 동작 | 이후 동작 |
|-----------|----------|----------|
| 1~5 | last_boxes + pad=5 | last_boxes + pad=5 (동일) |
| 6+ | `active=False` → 마스킹 **중단** | last_boxes + pad=8 (정적 마스킹 유지) |
| max_frame 초과 | (active=False 이후 무관) | max_frame 도달 시 자동 종료 |

**핵심 변경**:
- `active = False` 분기 **제거** → 마스킹이 절대 중단되지 않음
- CSRT 실패 6프레임 이상: 마지막 알려진 박스 위치(pad=8)로 계속 마스킹
- max_frame(다음 키프레임-1)까지 도달해야만 자동 종료
- 정적 모드 진입 시 1회 로그: `⚠️ CSRT 연속 실패 → 정적 마스킹 모드`

**보안 vs 화질 트레이드오프**:
- 장면 전환 후 카드 위치가 바뀌어도 이전 위치에서 계속 마스킹 적용됨
- 시각적으로 부자연스러울 수 있으나 PII 노출 원천 차단 (보안 우선)
- max_frame에 다음 키프레임이 새 정확한 위치로 마스킹 재개

## 12. 6~7차 코드 수정 — 잔상 깜빡임 및 장면 전환 즉시 감지

### 문제 (6→7→8초 구간 잔상 깜빡임)

**근본 원인 1 — ORB 인라이어 부족 시 아핀 변환 수락**:
- ORB 매칭 인라이어 수 검증 없이 아핀 행렬 적용 → 4개 매칭만으로도 계산 오류 발생
- 카드 회전 시 텍스트 박스가 실제 위치에서 벗어나는 현상

**근본 원인 2 — 누적 shift fallback 오차**:
- shift fallback이 `orig_boxes`(키프레임 초기 위치)에서 누적 이동량으로 계산
- 카드가 3D 회전할 때 텍스트의 실제 위치는 중심에서 방향이 바뀌는데, 누적 이동량은 이를 반영하지 못함
- → 회전이 진행될수록 박스 위치 오차가 커져 마스킹이 실제 텍스트를 못 덮음

**수정 (6차 → ORB 인라이어 검증)**:
- ORB 인라이어 ≥ 6개 + CSRT bbox 경계 검증 후에만 아핀 변환 수락
- 그 외: shift fallback으로 전환

**수정 (7차 → 증분 shift fallback)**:

| 항목 | 이전 (오류) | 이후 (수정) |
|------|------------|------------|
| shift 기준 | `orig_boxes` (키프레임 초기 위치) | `last_boxes` (직전 성공 프레임 위치) |
| 이동량 계산 | 키프레임 대비 누적 이동량 | 직전→현재 프레임 간 증분 이동량 |
| 회전 중 오차 | 회전 진행에 따라 누적 | 매 프레임 소량 보정 → 오차 최소 |

```python
# 이전 (누적 → 회전 시 오차 누적)
_dx = int((_tx + _tw/2) - state['orig_cx'])
_dy = int((_ty + _th/2) - state['orig_cy'])
_shifted = _shift_boxes(state['orig_boxes'], _dx, _dy)

# 이후 (증분 → 매 프레임 정확)
_last_cx = (state['last_bbox'][0] + state['last_bbox'][2]) / 2.0
_last_cy = (state['last_bbox'][1] + state['last_bbox'][3]) / 2.0
_inc_dx  = int((_tx + _tw / 2.0) - _last_cx)
_inc_dy  = int((_ty + _th / 2.0) - _last_cy)
_shifted = _shift_boxes(state['last_boxes'], _inc_dx, _inc_dy)
```
