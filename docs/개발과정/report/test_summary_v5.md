# 🏆 개인정보(PII) 비식별화 및 배경 복원 파이프라인 개발 종합 요약서 (v5)

이 문서는 프로젝트 초기 기획 및 요구사항 정의 단계부터, AnyText 시도 및 실패, 객체 탐지, 폰트 합성 시도 및 전면 폐기, **Stable Diffusion(디퓨전) 모델 도입과 3대 핵심 측정법을 결합한 하이브리드 배경 복원**, 그리고 **영상 비식별화 본격화 → OCR 엔진 PaddleOCR 전환 → 이미지·영상 코드 병합 → 서비스용 코드 분할(리포트/마스킹) → 상세페이지 오버레이 추적**까지의 전체 진화 과정을 흐름 순서로 누적 정리한 종합 보고서입니다.

> 📌 v5 누적 갱신 범위: v4(이미지 PII 마스킹까지) 이후 6/3~6/6 작업(Phase 10~15)을 추가했습니다. v4 본문은 역사 보존을 위해 그대로 두고, 이후 진화 내용을 이어서 기록합니다.

## 1. 📋 프로젝트 개요 및 목표

- **서비스 목적:** 개인정보 보호 자동화 서비스 구축.
- **최종 목표:** 신분증, 우편물, 택배송장 등 다양한 문서 이미지 내 개인정보(PII)를 정확하게 탐지하고, 주변 원본 텍스트 및 배경의 어떠한 훼손도 없이 감쪽같이 지우고 대체(배경 복원)하는 AI 자동화 파이프라인 구축.
- **최우선 타겟:** 실물 촬영 시 훼손 위험이 가장 높은 **'영수증', '택배송장', '신분증'** 3대 문서 이미지.
- **실행 환경:** Google Colab (GPU 사용), 모델 가중치는 Google Drive 마운트 경로(`/content/drive/MyDrive/final_PJ_model/`) 활용.

## 2. 🚀 전체 개발 진화 흐름 및 단계별 구현 내역

### 📍 [Phase 0] AnyText STE 최초 시도 및 실패 → 파이프라인 전환
- **시도:** AnyText(Scene Text Editor) 모델 하나로 OCR·배경 지우기·가짜 텍스트 합성을 일괄 처리 시도.
- **실패:** `expected scalar type Float but found Half` dtype 불일치 및 한국어 지원 한계(중국어/영어 위주).
- **결론:** AnyText 의존도를 낮추고, OCR·배경 복원 컴포넌트를 분리하는 하이브리드 파이프라인으로 전환.

### 📍 [Phase 1] OCR 엔진 선택 및 고도화
- **채택:** 한국어 정확도가 가장 높은 **Naver Clova OCR**을 메인으로, PaddleOCR을 폴백으로 채택.
- **캐싱:** 반복 API 호출에 따른 비용/시간 낭비를 막기 위해 좌표 결과를 JSON 파일로 캐싱 시스템 도입.
- *(※ 이후 Phase 12에서 비용·로컬화 이유로 PaddleOCR 한국어 단독으로 전환됨)*

### 📍 [Phase 2] YOLO + SAM2 객체 탐지 파이프라인
- **탐지:** YOLO 모델로 개인정보 존재 가능 영역(ROI) 1차 탐지 후, SAM2로 픽셀 단위 정밀 마스킹 진행. 탐지 실패 시 원본 전체 OCR 진행.

### 📍 [Phase 3] 텍스트 정밀 추출 및 PII 탐지 엔진
- **정규식 엔진:** 주민등록번호, 주소, 카드번호 등 핵심 개인정보 패턴 완벽 탐지.
- **2차 정제 필터:** 주소 중간 띄어쓰기나 숫자 혼합 시 탐욕적(Greedy)으로 쓸어 담은 후 정밀하게 걸러내는 2차 필터 구축.

### 📍 [Phase 4] 맥락 보존형 가짜 데이터(Fake Data) 생성 엔진
- **의미적 치환:** 단순 정규식 1:1 치환을 넘어, Gemini LLM을 연동하여 실제 존재하는 듯한 자연스러운 한국형 주소와 명칭 생성.
- *(※ 이후 합성 방향 폐기와 함께 현재 서비스 파이프라인에는 미사용 — '완벽한 제거+배경 복원'에 집중)*

### 📍 [Phase 5] 초정밀 픽셀 타겟팅 기반 배경 복원 (LaMa 인페인팅)
- **도입:** 단순 사각형 덮어쓰기 시 배경 무늬 훼손 방지를 위해 LaMa 모델 최초 채택.
- **이진화:** Otsu 이진화로 검은색 잉크 픽셀만 정밀 마스크로 추출 후 LaMa로 텍스트 부분만 복원 시도.

### 📍 [Phase 6] 폰트 매칭 및 3D 합성 시도 (6/1부로 전면 폐기)
- **시도:** 150종 폰트의 밀도와 장평을 수학적으로 매칭하고 `cv2.warpPerspective`로 3D 비틀기 합성을 시도.
- **결과 (폐기 결정):** 정교한 모델 없이는 폰트 매칭의 이질감이 커서 티가 많이 남을 인지. **결국 6/1 가짜 텍스트 '합성' 관련 코드는 전면 폐기하고, 오직 "완벽한 제거 및 배경 복원"에만 집중**하기로 방향 대폭 수정.

### 📍 [Phase 7] 물리적 고도화 패키지 도입
- **격자선 보호:** 모폴로지 기법으로 표의 가로/세로 격자선 및 경계 그림자를 100% 보존.
- **2중 격리 보호막:** 지우지 말아야 할 주변 원본 텍스트에 사방 3px 마진을 더해 훼손 0% 달성.

## 3. 🛡️ [핵심 보완] 디퓨전(SD) 도입 및 하이브리드 파이프라인 구축 (Phase 8 ~ 9)

6/1 합성 기능을 폐기한 직후, 가장 감쪽같은 배경 복원을 달성하기 위해 **Stable Diffusion 인페인팅 모델 도입**과 **다중 모델 결합**이라는 거대한 아키텍처 개편이 이루어졌습니다.

### 📍 [Phase 8] 디퓨전 모델 도입 및 3대 측정법 기반 하이브리드 분기
- **디퓨전(Stable Diffusion) 로컬 연동:** 코랩 로딩 시간 단축을 위해 SD 인페인팅 모델을 로컬 드라이브에 직접 다운로드하여 구축.
- **문제 발생:** SD 모델 단독 사용 시, 지정된 텍스트 범위 외의 이미지 전체가 훼손되거나 형태가 틀어지는 문제 발견.
- **3대 핵심 측정법 도입:** 배경을 정확히 구분하여 최적의 복원 모델을 투입하기 위해, ① **엣지 밀도 측정**, ② **픽셀 대비 및 분산 측정**, ③ **형태학적 구조선 감지** 기법을 적용한 하이브리드 자동 분기 시스템 설계.
- **하이브리드 교차 결합:** SD + LaMa + Seamless Clone을 혼합. (트러블슈팅 완료)
- SD 모델이 배경 생성 시 텍스트를 임의 생성하는 단점 → **LaMa + Seamless 만 사용**으로 결정.

### 📍 [Phase 9] 정밀 마스킹 튜닝
- **정밀 타겟팅 및 훼손 방지:** 주변 텍스트 훼손 없이 바운딩 박스 안에서만 정교하게 지우도록 마스킹 범위를 튜닝하여 인접 글자 보호 성공.

# 🎬 [v5 ] 영상 비식별화 본격화 → 서비스화 (Phase 10 ~ 15, 6/3~6/6)

이미지 파이프라인 안정화 후, 향후 계획이던 **영상 비식별화**를 실제 구현하고, 이어서 **OCR 엔진 전환·코드 병합·서비스용 코드 분할·상세페이지 오버레이**까지 진행했습니다.

**[전 구간 불변 보안 원칙]** 블러(모자이크)는 복원기로 되살릴 수 있어 금지. **모든 마스킹은 픽셀 비가역 삭제(`cv2.inpaint` Telea / LaMa)만** 사용.

### 📍 [Phase 10] 영상 비식별화 1차 — 증거 기반 마스킹 (6/3)
- 첫 영상 구현(선형 보간)에서 3대 문제 발견:
  ① OCR 키프레임 구간만 마스킹되고 사이 구간 PII 노출(고정 시간 임계로 보간 생략),
  ② 유령 잔상(LaMa 캐시 배경을 이동한 위치에 붙임),
  ③ PII가 사라진 뒤에도 마스킹 잔류(고정 1초 연장).
- 해결:
  - 고정 시간 임계 제거 → **OCR 부재 증거 기반 판단**(사이에 PII 미탐지 OCR 프레임 2개↑ 연속 → 이탈 확정, 그 외엔 보안 우선 마스킹).
  - LaMa 캐시 폐기 → **`cv2.inpaint`(Telea) 프레임 독립 처리**(유령 패치 구조적 제거).
  - 마지막 keyframe 이후도 부재 증거로 정확히 종료.

### 📍 [Phase 11] 영상 추적 고도화 — 픽셀 템플릿 추적(NCC) (6/3~6/4)
- CSRT·ORB 아핀 변환을 차례로 시도(잔상/회전 대응)했으나 불안정한 이력 → **단순·강건한 NCC 픽셀 추적으로 정착**.
- `_vid_match_shift`(`cv2.matchTemplate` TM_CCOEFF_NORMED): 이전 프레임 박스 픽셀을 다음 프레임 근방에서 매칭, **임계 0.5 미만이면 '동일 픽셀 없음 = PII 사라짐'으로 추적 중지**(과잉 마스킹 방지).
- `_vid_track_template`: 모든 OCR keyframe을 앵커로 **양방향 추적** + 첫 keyframe 이전 IoU **역소급(backfill)**.
- **부분 매칭 fallback**: 줌인 등으로 전체 매칭이 끊겨도, 긴 축 2등분 후 변하지 않은 절반(0.6~0.9)으로 추적 유지.
- **회전(affine) 추적**: 좌/우 절반이 모두 매칭되면 두 중심점으로 `estimateAffinePartial2D` 추정 → `_vid_affine_sane`(스케일 0.6~1.7, 회전 ±25°) 타당성 통과 시 polygon 회전 변환, 아니면 평행이동 fallback.
- **"한번 지우면 유지" 원칙** + 마스킹 대상은 매칭된 PII 박스만 타이트(배경/이름/유효기간 보존).

### 📍 [Phase 12] OCR 엔진 전환 및 최적화 — PaddleOCR 한국어 (6/4~6/5)
- **Naver Clova(API·비용) → PaddleOCR 한국어(PP-OCRv5, CPU) 단독 전환.**
- 실측으로 불필요 기능 제거: 영문 엔진 OFF(한글 엔진이 카드번호 동일 인식), deskew OFF(업스케일만으로 충분), textline-orientation OFF(21% 빠름). 샘플링 **1.0초**(픽셀추적이 사이 커버) + dHash 중복 프레임 제거.
- **작은 주소 미탐지** 개선: 중간 크기 라벨에서 1.3x로 안 잡히는 주소를 **조건부 고배율 재OCR(2.0x 순수 resize)** 로 탐지. 트리거 = (한글 문서성 OR 도로 단서) AND 행정구역 미완성 → 이미 잘 잡힌 프레임은 스킵해 속도 보전.

### 📍 [Phase 13] 적응형 인페인팅 + 이미지·영상 코드 병합 (6/5)
- **적응형 인페인팅 `inpaint_adaptive`**: 마스크 주변 배경 엣지밀도(`_bg_complexity`) 측정 → 단순 배경=Telea(빠름), 복잡 무늬 배경(신분증·지폐)=LaMa(AI 복원), LaMa 실패 시 Telea 자동 fallback.
- **코드 병합**(코드병합.md 기준): `OCR_Total.py`(이미지+영상 통합 — 이미지는 '프레임 1장'처럼 처리, JSON `frame_idx`/`timestamp_sec` 빈칸), `colab_pipeline_mask.py`(이미지+영상 통합). 구 Naver/영문/deskew/linescan 등 미사용 코드 정리.
- **PII 그룹화**: `(같은 zone OR bbox 공간 인접) + 같은 타입`을 Union-Find로 병합 → "주소1·카드번호1" 인스턴스 라벨/`pii_id` 부여. 단어로 쪼개진 한 주소를 하나로 통합.
- **리포트 데이터 JSON `report_{name}.json`**: 상세페이지 재표시용 좌표(pii_id·bbox·polygons·boxes·text 등) 보관.

### 📍 [Phase 14] 서비스화 — 코드 분할(리포트 / 마스킹) (6/6)
- 화면설계 기준 **무료 = 리포트 페이지까지 / 유료 = 마스킹·다운로드**. 흐름을 **1~4단계 / 5~7단계**로 분할(코드분할.md).
- **`colab_pipeline_report.py` (1~4단계)**: OCR(OCR_Total 병합) → PII 탐지 → 리포트(건수·박스·JSON·썸네일) → 선택. **원본 변형 없음**, torch/LaMa/inpaint 미로드(경량 로컬 CPU). 영상 report JSON에 **`keyframes` 앵커**(마스킹 추적용) 포함.
- **`colab_pipeline_mask.py` (5~7단계)**: report.py를 **import 하지 않는 완전 독립 코드**. 입력 = **원본 + report JSON + 선택 pii_id**. 마스킹 엔진·추적·워터마크·FFmpeg 자체 보유.
  - 5단계 미리보기(샘플 마스킹): 이미지=전체 마스킹 / 영상=탐지 지점 [±3초] 샘플 1클립 + 검토용 큰 워터마크 → 원본↔마스킹 슬라이더 비교.
  - 6~7단계 본 처리: 선택 PII만 전체 마스킹 → 워터마크 → 다운로드 파일(영상은 FFmpeg로 원본 오디오 유지).
- **워터마크**(예정 단계, 위치·방법 추후 논의): 가시(우하단 작은) + 비가시(휘도 스프레드 스펙트럼, 상관 검출로 출처 추적) + 미리보기 전용 큰 워터마크.

### 📍 [Phase 15] 상세페이지 영상 오버레이 추적 (6/6)
- 상세보기에서 **원본 영상 재생(인코딩 0) 위에 PII 박스가 텍스트를 따라 이동**(객체추적 데모 방식, 브라우저 Canvas 오버레이).

## 6. ⚠️ (현재) 한계점 및 향후 계획

1. **분할 후 런타임 통합 테스트 예정** — report→mask 분리 코드는 구문 검사까지 완료. 실제 파일로 리포트 생성 → 미리보기 → 본 마스킹까지 1회 점검 필요(Colab/GPU).
2. **워터마크 위치·방법 확정 필요** — 현재 기본 구현(가시 우하단 + 비가시 스프레드 스펙트럼). 강건성 필요 시 blind watermark 라이브러리 검토.
3. **영상 내 같은 타입 다중 PII 분리** — 현재 영상은 타입별 1 인스턴스 정책. 한 영상에 같은 타입이 공간적으로 여러 개일 때 위치 클러스터링 분리는 추후 고도화.
4. **STT(음성→자막) 연동** — 별도 담당자 개발 중. 업로드→결과까지 총 시간 최소화를 위해 OCR/마스킹 워커와 STT 워커 병렬(A안) 권장.
5. **극단 줌/회전 구간** — NCC 임계 0.5 미만으로 변형되는 극단 구간은 추적 불가(보안 우선 허용 한계).
