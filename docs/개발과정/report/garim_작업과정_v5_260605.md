# 작업 A — 조건부 고배율 재OCR (작게 보이는 주소 탐지)
**대상**: `Paddle_OCR_vidio.py`

## A-1. 배경
frame 245(10.21초)처럼 라벨이 중간 크기로 보이는 구간에서 주소가 너무 작아
1.3x로는 글자조차 안 잡힘 → 주소 keyframe 미생성 → 마스킹 누락(노출).

## A-2. 변경 내용
- 설정: `OCR_HIRES_RETRY_ENABLED=True`, `OCR_HIRES_SCALE=2.0`, `OCR_HIRES_BOX_MIN=8`, `OCR_HIRES_KR_MIN=2`
- `enhance_frame_for_ocr(frame, scale=None)` — 배율 인자화
- `_to_boxes(ocr_raw, scale=None)` — 좌표 역변환 배율 인자화
- `_merge_hires(base, hires)` 신설 — **고배율 우선** 병합(겹치면 긴 텍스트/높은 신뢰도 채택).
  기존 `_merge_ocr_results`는 숫자 우선이라 한글 주소가 깨진 숫자에 밀리는 문제 해결
- `process_frame` 조건부 재OCR:
  - 트리거: **(한글 문서성 OR 도로 단서) AND 행정구역(시+구) 미완성**
    → 주소 이미 잘 잡힌 프레임은 스킵(속도 절감)
  - 재OCR은 **순수 업스케일(cv2.resize)만** 사용
    ※ CLAHE/언샤프(enhance)는 작은 주소를 과강조해 오인식
    (실측: enhance → `Mn구O일`, 순수 resize → `서표제명시 영동교구 영순로10일`)

## A-3. 실측 (배율별 frame 245)
| 배율 | 주소 탐지 |
|------|----------|
| 1.3x | `3122(영37)` ❌ |
| 1.5x | `312(영용377)` ❌ |
| **2.0x (순수 resize)** | **`서표제명시 영동교구 영순로10일`** ✅ |

## A-4. 검증 (송장 영상)
- 주소 keyframe: [45, 69, 85, 154, 179, **245**, 263] — frame 245 포함
- **frame 245 마스킹: True** ✅
- 속도: 송장 37초 → **70초** (작은 주소 잡는 대가. 행정구역 온전 프레임 스킵으로 87→70 단축).
  카드 등 한글 적은 영상은 재OCR 거의 없음 → 속도 영향 미미

# 작업 B — 회전 추적 (각도 급변 구간 배경 침범 해결)
**대상**: `colab_pipeline_video.py`

## B-1. 배경
카메라가 빠르게 움직여 라벨 각도가 급변하면, 부분 매칭의 **평행이동만**으로는
polygon이 안 따라 돌아 마스킹이 옆 배경(파란 비닐)으로 넘어감.

## B-2. 변경 내용
- `_vid_affine_sane(M)` 신설 — 추정 변환 타당성 검사(스케일 0.6~1.7, 회전 ±25°). 과회전/과확대 거부
- `_vid_match_shift` 확장 — 반환 형식 `('shift',dx,dy)` / `('affine',M)` / None:
  - 부분 매칭 **좌/우 두 절반이 모두 매칭**되면 두 중심점으로
    `cv2.estimateAffinePartial2D`로 회전·크기(similarity) 추정 → `('affine', M)`
  - 변환이 비정상이면 평행이동 fallback
  - 한쪽만 매칭 → 평행이동
- `_vid_track_run` 분기 — affine이면 `_transform_boxes_affine`으로 polygon 회전 변환,
  shift면 기존 평행이동. bbox는 변환된 박스에서 재계산

## B-3. 검증 (송장 영상)
- 추적 중 **affine(회전) 8회 / shift(평행) 190회** 발동 — 각도 급변 구간에서만 회전 적용
- frame 168(7초 각도 급변): 기울어진 라벨의 주소 삭제, **배경 침범 없음** ✅

# 작업 D — 환경설정 일원화 + OCR↔마스킹 통합 실행 + 그룹화 보정

## D-1. 환경설정 일원화 (경로는 colab 한 곳에서만)
- **OCR_Total.py = OCR 옵션 전용 모듈**: 경로/테스트경로/`__main__` 전부 제거.
  colab이 import해 함수(`process_image`/`process_video`)로 호출하며 **경로는 colab이 인자로 전달**.
  → OCR은 성능 옵션만 두고, 경로는 건드릴 필요 없음.
- **colab_pipeline_mask.py = 경로 단일 관리**: 맨 위 '환경설정'에 경로/파라미터/테스트경로 집약
  (BASE_DIR 자동판별, INPUT_*/OCR_*/OUTPUT_DIR/FONT_DIR, VID_*/FFMPEG_*/MASK_* 등).
  함수 내부 하드코딩 경로 전부 제거 → 서비스 연동 시 colab만 수정.
- `os.environ` 우회 2곳(PIR/oneDNN, KMP_DUPLICATE_LIB_OK)에 **이유+서비스시 제거 권장** 주석.

## D-2. OCR → 마스킹 통합 실행 (실서비스 흐름)
- colab이 `import OCR_Total` (같은 폴더). `run_pipeline(경로)`가 **OCR 먼저 → 마스킹** 순차 실행.
- `run_pipeline`은 **파일/폴더 모두** 처리(폴더는 내부 이미지·영상 일괄). `_run_one`이 단일 처리.
- 코랩 권장 실행: **import 방식**(아래). exec(open().read())는 노트북 `__name__='__main__'`이라
  테스트 블록이 자동 실행돼 중복 처리됨 → import 권장.
  ```python
  import sys; sys.path.insert(0,"/content/drive/MyDrive/final_PJ_model")
  import importlib, colab_pipeline_mask as M; importlib.reload(M)
  M.run_pipeline(".../파일 또는 폴더")
  ```
- LaMa 미설치 시 매 프레임 재시도하던 버그 수정(`lama_model=False` 캐싱 → 1회만 시도 후 Telea).
