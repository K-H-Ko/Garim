## 1. DB 초기화 문제 해결 (Docker PostgreSQL)

### 문제
기존 Docker 볼륨이 살아있어 `init SQL`이 재실행되지 않음 → `detections` 테이블 등 전체 스키마 없음.
```
ERROR: relation "detections" does not exist
```

### 원인
PostgreSQL init 스크립트(`/docker-entrypoint-initdb.d/`)는 볼륨이 **처음 생성될 때만** 실행됨.  
`docker-compose down` 후 `up`을 해도 볼륨이 남아있으면 스크립트 재실행 안 됨.

### 해결 방법 (PowerShell — Bash 경로 변환 문제 우회)
```powershell
# 1. SQL 파일 컨테이너 내부로 복사
docker cp "docker/database/init/0_init_table_v11.sql" final_db:/tmp/
docker cp "docker/database/init/seed_users.sql" final_db:/tmp/
docker cp "docker/database/init/seed_payments.sql" final_db:/tmp/
docker cp "docker/database/init/2_add_pii_id_to_detections.sql" final_db:/tmp/
docker cp "docker/database/init/3_add_pii_id_index.sql" final_db:/tmp/

# 2. 순서대로 실행
docker exec -it final_db psql -U 1team -d 1team -f /tmp/0_init_table_v11.sql
docker exec -it final_db psql -U 1team -d 1team -f /tmp/seed_users.sql
docker exec -it final_db psql -U 1team -d 1team -f /tmp/seed_payments.sql
...
```

### seed_users.sql 버그 수정
**파일**: `docker/database/init/seed_users.sql`

| 오류 | 원인 | 수정 |
|------|------|------|
| `column "is_active" does not exist` | v11 스키마에서 `is_active` → `status`로 컬럼명 변경 | `is_active=true` → `status='active'` |
| `relation "risk_reports" does not exist` | v11에서 테이블 삭제 | INSERT INTO risk_reports 블록 전체 제거 |
| plans 중복 INSERT 오류 | 루프에서 중복 발생 | `ON CONFLICT (plan_code) DO NOTHING` 추가 |

### 결과
- **35개 테이블** 생성 완료
- **100명 유저** 시드 데이터 삽입 완료
- `mask_preview`, `mask_final` job_type CHECK constraint 적용 완료
- `detections.pii_id` 컬럼 추가 완료 (timeline_markers 매핑용)

## 3. 로컬 워커 자동화 — `ocr_mask/local_worker.py` 신규 생성

### 목적
사용자가 파일 업로드 → 백엔드에 Job 생성 → **자동으로 OCR 분석 + 마스킹까지 처리**.  
상시 실행해두면 수동 개입 없이 파이프라인 전 과정 자동화.

### 처리 흐름
```
[메인 루프 — 5초 폴링]
GET /worker/jobs/next
  │
  ├─ job_type == 'analysis'
  │    ① POST /worker/jobs/{id}/accept
  │    ② OCR_pipeline_report.run_pipeline(file_path) → index.json 생성
  │    ③ backend_json_merger.merge_visual_and_audio_reports() → result.json
  │    ④ POST /worker/jobs/{id}/results/pii      (visual_pii 등록)
  │    ⑤ POST /worker/jobs/{id}/results/artifact (3종: result.json, 상세보기.mp4, tracks.json)
  │    ⑥ POST /worker/jobs/{id}/complete
  │
  └─ job_type == 'mask_preview' | 'mask_final'
       ① POST /worker/jobs/{id}/accept
       ② POST /worker/jobs/{id}/results/processed-file (더미 — 원본 경로 등록)
       ③ POST /worker/jobs/{id}/complete
```

### 실행 방법
```bash
# 프로젝트 루트에서
python ocr_mask/local_worker.py

# 환경변수 (기본값으로 동작)
WORKER_BASE=http://localhost:8000
WORKER_API_KEY=change_me_to_a_long_random_secret
POLL_INTERVAL_SEC=5
```

### 주요 구현 포인트
| 항목 | 내용 |
|------|------|
| `/worker/jobs/next` 응답 파싱 | `data.get("job")` — 응답이 `{"job": {...}}` 또는 `{"job": null}` 형태 |
| STT 없이 병합 | `audio_pii_segments=[]` 빈 배열로 OCR만으로 병합 |
| 더미 마스킹 | 실제 GPU 마스킹 대신 원본 파일 경로를 `processed_files`에 등록 |
| artifact 파일 없음 처리 | `_register_artifacts()`에서 파일 존재 확인 후 없으면 건너뜀 |
| 인증 | `Authorization: Bearer {WORKER_API_KEY}` 헤더 |
