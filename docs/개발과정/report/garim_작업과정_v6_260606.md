# 작업 G — `colab_pipeline_mask.py` (완전 독립)

## G-1. 엔진 자체 보유 (self-contained)
- report.py 에서 떼어낸 엔진을 이 파일이 **자체 정의**:
  적응형 인페인팅(`inpaint_adaptive`/`_lama_inpaint`/`_bg_complexity`),
  이미지 마스킹(`synthesize_pii_region`/`mask_selected_on_image`),
  영상 픽셀추적(`_vid_ncc_probe`/`_vid_match_shift`/`_vid_track_run`/`_vid_track_template` 등),
  워터마크, FFmpeg. LaMa/torch 셋업도 이 파일에.

## G-2. 미리보기(샘플 마스킹) `run_preview`
- 이미지: (단일) 전체 마스킹 후 미리보기 / 다중이면 1개 샘플(호출측 선택)
- 영상: 개인정보 **탐지 지점 [이전 3초 ~ 이후 3초] 샘플 1개** 클립만 마스킹(+해당 구간 오디오 동기)
- 미리보기엔 **검토용 큰 워터마크**(대각선 반복) + 비가시 워터마크 → 원본↔마스킹 슬라이더 비교용

## G-3. 본 처리 `run_masking`
- 선택 `pii_id` 만 전체 마스킹:
  - 이미지 `mask_image_full` → `mask_selected_on_image`
  - 영상 `mask_video_full` → JSON `keyframes`로 `_vid_track_template` 추적 → 프레임 인페인팅
- 워터마크(본 처리물): 우하단 작은 가시 + 비가시(휘도 스프레드 스펙트럼)
- 영상은 FFmpeg 로 원본 오디오 유지하여 `결과_{name}.mp4`(다운로드용) 출력

## G-4. 워터마크 (예정 단계 — 위치/방법 추후 논의)
- `add_visible_watermark`(우하단), `embed_invisible_watermark`(휘도 패턴),
  `verify_invisible_watermark`(상관 검출 출처 추적), `add_preview_watermark`(검토용 큰 워터마크)
- payload 시드 = `WATERMARK_PAYLOAD`(서비스에선 사용자/잡 ID 권장). 기본 구현 → 차후 robust 라이브러리 교체 가능

# 작업 H — 상세페이지 영상 오버레이 추적 (원본 재생 + 박스가 텍스트 따라 이동)

## H-1. 결정 배경
- 화면설계의 상세보기: 타임라인에 PII 위치 표시, 클릭 시 해당 장면 이동, 재생 중 박스가 PII 따라 이동.
- 오해 정정: **상세페이지 영상 재생엔 인코딩이 없다**(원본 그대로 스트리밍). 인코딩은 미리보기/본처리뿐.
- 움직이는 박스 = 마스킹 합성이 아니라 **원본 영상 위 Canvas 오버레이**(YOLO 데모 방식).
- 선택지: A(키프레임 보간, 추가비용 0) vs A+(정밀 추적, 디코딩 1패스). 사용자 선택 = **A+(부드러운 픽셀추적)**.
  - 근거: A+는 인코딩이 아니라 디코딩+matchTemplate → 마스킹의 1/5~1/10, PII 등장 구간만 추적해 가벼움.

## H-2. 구현 (report.py — 가벼운 cv2 추적, torch/LaMa/inpaint 없음)
- 추가 함수: `_transform_boxes_affine`, `_vid_affine_sane`, `_vid_ncc_probe`, `_vid_match_shift`,
  `_bbox_quad_from_boxes`, `_track_one_dir`, `_build_overlay_tracks`
- `analyze_video` 가 그룹 빌드 후 `_build_overlay_tracks` 호출 → 그룹별 프레임 좌표 track 생성
- 설정 재추가: `TRACK_FPS`(기본 10, 다운샘플), `VID_MATCH_THRESH/SEARCH_RATIO/EDGE_MAX_SEC/AFFINE_*`

## H-3. report JSON 추가 필드 (`pii_groups[].track`)
```
track: [ {sec, frame, bbox:[x1,y1,x2,y2], quad:[[x,y]*4]}, ... ]   // 영상만, TRACK_FPS로 다운샘플
```
- `bbox`(축 정렬) + `quad`(회전 사각형 — 텍스트 기울기 반영, "객체추적" 느낌). 앵커(OCR keyframe)는 항상 포함.

## H-4. 프론트 연동(요약, 코드 주석에도 명시)
1. `<video>` 원본 재생(인코딩 0)
2. Canvas 가 `video.currentTime`(초)로 `track` 앞뒤 점 **선형보간** → `quad` 그림
3. 타임라인 마커 = `track`/`keyframes` 의 `sec`, 클릭 → `video.currentTime = sec`(즉시 점프)
