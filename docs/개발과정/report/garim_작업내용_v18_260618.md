## 📌 세션 6: 코랩 이미지 마스킹 실패 수정

### 증상
```
ERROR:garim-worker:job 처리 실패:
OpenCV(4.10.0) .../loadsave.cpp:1141: error: (-2:Unspecified error)
could not find encoder for the specified extension in function 'imencode'
```
- 이미지 마스킹(`mask_final`) 시 OpenCV 에러 발생 → 프론트엔드에 "처리 실패" 표시

### 근본 원인
- 코랩 워커에서 원본 파일 다운로드 시 `Content-Disposition` 헤더의 한글 파일명(`filename*=UTF-8''...` 형태)을 파싱 실패
- 파일이 확장자 없이 저장됨 (`upload_173b080b-...`)
- `mask_image_full`에서 `cv2.imencode(img_path.suffix, masked)` 호출 시 `suffix = ''` → 인코더 찾기 실패

### 수정
| 파일 | 수정 내용 |
|------|-----------|
| `colab/garim_colab_worker.py` `download_file()` | `filename*=UTF-8''...` 형태 파싱 지원 + 확장자 누락 시 `Content-Type` 헤더로 추론 |
| `ocr_mask/colab_pipeline_mask.py` `mask_image_full()` | `img_path.suffix`가 빈 문자열이면 `.jpg`로 폴백 (안전장치) |

## 📌 세션 7: OCR 주소 탐지 정확도 개선 (사용자 직접 수정 포함)

### 문제
- 택배송장: 주소가 원래 잡히지 않아야 하는데 잡힘 (오탐)
- 우편물3: 주소 일부만 잡히고 전체가 누락됨 (미탐)

### 사용자 직접 수정 (`OCR_Setting.py`)
| 파라미터 | 변경 전 | 변경 후 | 의도 |
|----------|---------|---------|------|
| `GAP_MULT` | 3.0 | 1.5 | 클러스터 간격 기준을 좁혀 분리 강화 |
| `BG_COLOR_THRESH` | 22.0 | 26.0 | 배경색 구분 임계값 확대 |
| `COMPLEXITY_THRESH` | 0.06 | 0.07 | 복잡도 구분 임계값 확대 |
| `COMPLEXITY_PAD_RATIO` | 1.0 | 1.3 | 복잡도 샘플 영역 확장 |
- 시각화 디버깅 코드(region jpg, text txt) 임시 활성화 (테스트 후 재비활성화 예정)

### 에이전트 수정 (`OCR_pipeline_report.py`)
- **주소 정규식 분기 2**: `(?:[가-힣]{1,4}(?:구|군|시)[\s,]+[가-힣\s]{0,8})?` 접두 패턴 추가
  - OCR이 행정구역을 깨뜨려 잡아도 뒤의 '로+길+숫자' 코어와 함께 매칭되도록
- **`build_full_text()` 토큰 결합 로직**: 간격이 평균 글자 너비의 0.5배 미만이면 공백 없이 붙이는 `tight` 조건 추가
  - `경북영덕` + `군영해면` → `경북영덕군영해면`으로 이어져 주소 정규식 매칭 가능하게
