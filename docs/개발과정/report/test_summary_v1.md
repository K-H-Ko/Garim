# 개인정보(PII) 비식별화 및 합성 텍스트 파이프라인 개발 요약

## 1. 프로젝트 개요
- **목표:** 신분증, 우편물, 택배송장 등의 문서 이미지에서 개인정보를 탐지하고, 원본 배경의 훼손 없이 가짜(합성) 데이터로 자연스럽게 교체하는 AI 파이프라인 구축.
- **핵심 기술:** Google Cloud Vision OCR, 정규표현식(Regex), Gemini LLM(가짜 데이터 생성), LaMa(배경 인페인팅), PIL & OpenCV(텍스트 렌더링 및 3D 공간 합성).

## 2. 주요 개발 단계 및 핵심 구현 내역

### [Phase 1] 데이터 추출 및 탐지 로직 구축
- **Google Cloud Vision 연동:** 이미지 내 텍스트와 좌표(Bounding Box)를 단어/줄 단위로 정밀하게 추출 및 군집화.
- **정규표현식(Regex) 엔진 개발:** 주민등록번호, 전화번호, 주소, 카드번호, 계좌번호, 이메일, 여권번호 등 7대 개인정보 탐지 패턴 구축.
- **2차 필터링(Post-processing):** OCR이 잘못 읽은 쓰레기 텍스트(예: 영문, 무의미한 단어)를 걸러내고, 유효한 주소 접미사(길, 로, 동)와 숫자만 남기도록 정제 로직 고도화.

### [Phase 2] 가짜 데이터(Fake Data) 생성 및 템플릿 엔진화
- **정규식 기반 1:1 구조 치환:** 탐지된 원본 데이터의 구조(글자 길이, 하이픈 등 기호 위치)를 그대로 유지하면서 무작위 가짜 데이터 생성.
- **주소 전용 치환 엔진:** `X시 X구 ZZZ길`과 같이 한글은 무작위 한글 단어로, 숫자는 숫자로 치환하며, '단위어(시/구/동/호)'는 보존하여 이질감 없는 사실적인 가짜 주소 생성.

### [Phase 3] 텍스트 마스킹 및 배경 복원 (LaMa)
- **형태학적 잉크 마스킹:** 단순히 네모난 박스로 지우는 대신, OpenCV의 이진화 및 팽창(Dilation) 기법을 사용해 글씨의 잉크 픽셀 범위만 추적하여 마스킹 시도.
- **LaMa 인페인팅 적용:** 마스킹된 텍스트 영역을 주변 배경색과 패턴을 참조하여 지워진 흔적 없이 복원.

### [Phase 4] 동적 폰트 스캔 매칭 및 텍스트 3D 합성
- **원본 글씨 특징 스캔:** 원본 이미지에서 폰트의 굵기(Stroke Thickness), 잉크 색상(RGB), 장평(Aspect Ratio), 폰트 계열(Serif/Sans-serif), 선명도 추출.
- **150종 폰트 정밀 수학적 매칭:** 사용자가 제공한 150여 개의 폰트(TTF, OTF, TTC) 전체의 '밀도'와 '글자당 장평'을 캐싱한 뒤, 원본 텍스트와 수학적 오차가 가장 0에 가까운 폰트를 자동으로 선발.
- **텍스처(질감) 및 3D 왜곡 합성 (Warping):** 
  - 노이즈 및 가우시안 블러를 통해 실제 종이에 잉크가 번진 듯한 리얼 질감 구현.
  - `cv2.warpPerspective`를 사용하여 원본 텍스트의 3D 공간 다각형 좌표에 맞춰 가짜 텍스트를 비틀어 배치.

## 3. 사용자 피드백을 통한 트러블슈팅 및 개선 내역
- **주소 토막남 현상 해결:** 주소 중간에 띄어쓰기나 숫자가 있을 때 정규식이 끊기는 문제를 해결하기 위해, 탐욕적(Greedy) 방식으로 한 줄의 텍스트를 넓게 쓸어 담은 뒤 파이썬 코드로 유효성을 정밀 필터링하도록 전면 개편.
- **폰트 고정 출력 오해(Sans-Regular) 해결:** 확장자 필터 문제로 .otf 폰트 등이 무시되던 현상을 고치고, 코드 로그에 실제로 150개 중 선택된 '진짜 폰트 파일명'을 명확히 출력하도록 수정.
- **주소 템플릿 붕괴 버그 수정:** 2차 주소 정제 후 가짜 텍스트 재생성 시, 치환 템플릿(X, Z) 로직이 무시되어 길이가 맞지 않는 주소가 생성되던 치명적 버그 수정.

## 4. 현재 직면한 한계점 및 다음 스텝 예정

1. **얼룩(배경 찌꺼기) 완벽 제거:** 기존의 '잉크 픽셀'만 추적해서 마스킹하는 방식은 안티앨리어싱 찌꺼기를 남김. ➡ 개별 단어의 **바운딩 박스(사각형) 전체를 통째로 마스킹**하여 배경을 흔적 없이 완벽히 날려버리는 방식으로 교체.
2. **주소 합침/분리 정밀도 극대화:** ➡ 중간에 주소와 무관한 완전히 독립된 숫자 덩어리나 영문이 등장하여 주소의 흐름이 끊기면, **그 즉시 탐지를 중단(Truncate)** 하도록 2차 필터 강화.
3. **폰트 이질감(가짜 느낌) 완벽 제거:** 긴 문장을 일직선으로 그려 통째로 잡아당기던(Warping) 방식은 폰트가 일그러지는 주범임. ➡ 가짜 텍스트를 단어별로 쪼갠 뒤, 원본 단어 개개의 고유한 높낮이와 기울기 박스에 맞춰 **단어 하나씩 개별적으로 합성**하여 실제 인쇄물 수준의 퀄리티 달성.
