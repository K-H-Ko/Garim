# 개인정보(PII) 비식별화 및 합성 텍스트 파이프라인 개발 요약 (종합본)

이 문서는 프로젝트 초기 요구사항 분석 단계부터 패키지 충돌 트러블슈팅, 최신 AI 모델(YOLO, SAM 2, AnyText) 도입까지의 전체 개발 과정을 시간순 및 기능별로 핵심만 요약한 문서입니다.

---

## 1. 프로젝트 초기 기획 및 요구사항 분석
- **목표 정의:** 요구사항 명세서(ID 01 ~ 13, FR-17)를 기반으로, 문서 이미지 내 개인정보를 탐지하고 원본 훼손 없이 가짜 데이터로 치환하는 AI 파이프라인 구축.
- **초기 타겟 선정:** 전체 데이터가 아닌 **'영수증'과 '택배송장'** 두 가지 카테고리 이미지를 최우선 테스트 대상으로 선정하여 기능 검증 돌입.
- **아키텍처 설계:** Gemini LLM(3.1 Flash-Lite)을 활용해 메타데이터(개인정보)를 맥락에 맞는 가짜 데이터로 치환하기로 설계하고, 코랩 환경에서 작동할 초기 파이썬 파이프라인(`colab_pipeline.py`) 코드베이스 구축 시작.

---

## 2. 주요 개발 단계 및 핵심 구현 내역 (진화 과정)

### [Phase 1] 텍스트 추출 및 개인정보 탐지 (규칙 기반 → AI 기반)
- **초기 시도 (Google Cloud Vision):** 단어/줄 단위 좌표를 정밀 추출.
- **최종 도입 (EasyOCR):** 외부 API 의존성을 줄이고, 로컬 환경에서 유연하게 작동하는 오픈소스 텍스트 추출로 전환.
- **정규표현식(Regex) 엔진 고도화:** 주민번호, 전화번호, 주소 등 주요 패턴 탐지. 주소 중간에 띄어쓰기나 무관한 숫자가 끼어들어 텍스트가 파편화되는 현상을 막기 위해 탐욕적(Greedy) 탐색 후 정밀 필터링하는 '2차 정제 로직' 추가.

### [Phase 2] 가짜 데이터(Fake Data) 생성 엔진
- **구조 보존형 치환:** 탐지된 원본 데이터의 구조(글자 길이, 하이픈 등 기호 위치)를 그대로 유지.
- **Gemini LLM 연동:** 단순 무작위 문자가 아니라, 의미와 맥락을 파악해 이질감 없는 진짜 같은 가짜 주소 및 텍스트를 생성 (예: `서울시 강남구` ➡ `푸른동 맑은길`). 보안을 위해 API Key 입력부는 사용자가 직접 기입하도록 유연하게 설계.

### [Phase 3] 문서 객체 탐지 및 정밀 마스킹 (YOLO + SAM 2 도입)
- **초기 픽셀 추적의 한계:** 글씨의 잉크 픽셀만 추적해 지우는 방식은 가장자리에 안티앨리어싱 찌꺼기(얼룩)를 남기는 문제가 있었음.
- **YOLOv8 기반 객체 인식:** 영수증, 택배송장 등 타겟 문서 객체만 정확히 인식하고 크롭(Crop)하여 불필요한 노이즈 사전 차단.
- **SAM 2 (Segment Anything 2) 마스킹:** 찌꺼기 문제를 완벽히 해결하기 위해, SAM 2를 이용해 텍스트가 포함된 영역을 픽셀 단위로 정밀하게 분할(Segmentation)하여 배경 복원을 위한 완벽한 마스크 생성.

### [Phase 4] 배경 복원 및 텍스트 렌더링 (LaMa ➡ AnyText 진화)
- **초기 시도 (LaMa + PIL 3D Warping):** LaMa로 글씨를 지운 후, 150종의 폰트 중 밀도/장평 오차가 가장 적은 폰트를 수학적으로 매칭해 `cv2.warpPerspective`로 3D 비틀기 합성 진행.
  - *한계점:* 긴 문장 렌더링 시 일직선으로 잡아당겨져 폰트가 심하게 일그러지는 이질감(가짜 느낌) 발생.
- **최종 해결 (AnyText 도입):** 수동적인 3D 렌더링의 한계를 극복하기 위해, Diffusion(ControlNet) 기반의 텍스트 합성 AI인 **AnyText**를 전격 도입. 원본 배경의 질감, 잉크 번짐, 색상, 조명, 기울기를 모델이 스스로 분석하여 감쪽같이 합성.

## 3. 핵심 트러블슈팅 및 패키지 충돌 해결 (Colab 환경 극복)
코랩(Python 3.12) 환경에서 구버전 AI 모델(AnyText, LaMa 등)을 구동하기 위해 발생한 극심한 의존성(Dependency) 충돌을 패키지 레벨에서 완벽히 해결함.

1. **Rust 컴파일 및 Tokenizers 에러:** 
   - 최신 파이썬 환경에서 `transformers 4.26.1`이 요구하는 구버전 `tokenizers(0.13.3)`가 억지로 빌드되려다 실패하는 에러. 스크립트 단에서 버전을 속이는 몽키 패치 적용.
2. **Huggingface Hub 상수 삭제 충돌:** 
   - `diffusers 0.14.0`이 최신 `huggingface_hub`에서 삭제된 과거의 캐시 상수(`hf_cache_home`) 및 `cached_download`를 호출하여 발생하는 `ImportError` 해결. 
   - 모든 패키지(`transformers`, `diffusers`, `modelscope`)와 완벽히 호환되는 **"황금 버전(huggingface_hub==0.22.2)"**을 발굴하여 스크립트 실행 시 자동 다운그레이드 되도록 적용.
3. **AnyText FP16 / FP32 혼합 정밀도 충돌:** 
   - AnyText 추론 중 DDIM Sampler에서 `expected scalar type Float but found Half` (PyTorch 데이터 타입 충돌) 에러 발생. 
   - GPU 메모리 최적화를 위해 켜둔 `use_fp16` 플래그를 `False`로 비활성화하여, 표준 FP32 단일 정밀도로 강제 구동시켜 원천 차단.

## 4. 결론 및 향후 계획
- 요구사항 명세서를 바탕으로 시작한 프로젝트가 한계(수학적 렌더링 이질감, 배경 얼룩 등)에 부딪힐 때마다, **YOLO, SAM 2, AnyText 등 최첨단 Vision AI 모델을 순차적으로 융합하며 완전 자동화 파이프라인**으로 성공적으로 진화함.
- 영수증과 택배송장에서의 완벽한 기능 검증(정상 탐지 → LLM 치환 → 자연스러운 파일 추출)이 마무리되었으므로, 차후 전체 데이터셋 및 영상 데이터로 확장을 진행할 수 있는 견고한 토대를 마련함.
