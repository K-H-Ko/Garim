### 2-5. DB CHECK 제약 조건 수정

- 오류: `(psycopg2.errors.CheckViolation) new row for relation "analysis_jobs" violates check constraint "ck_analysis_jobs_job_type_v4"`
- 원인: `job_type` 허용 목록에 `stt_analysis` 미포함
- 해결:
  1. `docker/database/init/4_add_stt_analysis_jobtype.sql` 마이그레이션 파일 생성
  2. 실행 중인 DB에 즉시 적용 — `ck_analysis_jobs_job_type_v4`, `ck_analysis_jobs_job_type` 두 제약 모두 삭제 후 재생성

```sql
ALTER TABLE analysis_jobs
    ADD CONSTRAINT ck_analysis_jobs_job_type CHECK (
        job_type IN ('analysis','preview','render','sns_scan',
                     'whitelist_scan','mask_preview','mask_final','stt_analysis')
    );
```

## 4. 남은 과제 (프론트-백엔드 연결)

### 4-1. 상세보기 페이지 (`AnalysisDetail.jsx`)
- **현황**: 페이지 컴포넌트 존재, API 연결 미완
- **해야 할 것**:
  - `GET /analysis/jobs/{job_id}/result` — result.json 데이터 로드
  - `pii_groups` (시각 PII) + `audio_pii_groups` (음성 PII) 탭 구분 표시
  - `timeline_markers` 기반 타임라인 UI 연결
  - `is_selected` 체크박스 → `PATCH /analysis/jobs/{job_id}/pii/{pii_id}` 연결

### 4-2. 리포트 페이지 (분석 완료 후 최종 결과 화면)
- **현황**: result.json 생성은 완료, 프론트 미연결
- **해야 할 것**:
  - `visual_pii_count`, `audio_pii_count`, `total_pii_count` 요약 카드
  - PII 목록 테이블 (시각/음성 구분)
  - 마스킹 선택 → 마스킹 시작 버튼 플로우

### 4-3. 마스킹 결과 다운로드
- **현황**: `mask_preview`, `mask_final` job은 더미 처리 중
- **해야 할 것**: 실제 마스킹 파이프라인 연결 또는 더미 → 완성본 교체

### 4-4. Cloudflare 터널 연결 안정화
- **현황**: 터널 재시작 시 URL이 바뀌어 Colab 워커 재연결 필요
- **개선 방향**: 백엔드 URL을 환경변수/설정 파일로 분리하거나, 고정 터널 도메인 사용

### 4-5. 전체 파이프라인 end-to-end 검증
- 영상 업로드 → analysis + stt_analysis 병렬 실행 → 병합 → result.json 생성 → 상세보기 전 과정 실제 테스트 완료 필요

## 5. 현재 시스템 흐름 요약

```
[영상 업로드]
      │
      ├─► [로컬 워커] analysis job
      │     1. 프레임 OCR 실행 → {stem}_index.json
      │     2. Colab STT 완료 대기 (최대 5분)
      │     3. merge → {stem}_result.json
      │     4. index.json + stt.json 삭제
      │     5. PII/artifact 백엔드 등록 → 완료
      │
      └─► [Colab 워커] stt_analysis job
            1. 음성 추출 (WAV)
            2. STT Whisper 실행
            3. PII 감지
            4. {stem}_stt.json 저장
            5. audio_pii_segments 백엔드 등록 → 완료
```
