# 작업 내용 v1 (2026-06-13)

## 주제: 프론트/백엔드 ↔ 파이프라인 연동을 위한 프로젝트 v2 청사진 문서 작성

OCR·영상추출 파이프라인 검증 완료 후, **기존 프론트/백엔드 구조에 우리 파이프라인
(`ocr_mask/`)을 끼워 넣어 연동**하기 위한 기준 문서(v2 2종)를 작성.

---

## 1. 전체 프로젝트 재파악 (탐색)

프론트·백엔드 일부 업데이트 반영 위해 코드/문서 전수 파악.

### 파악 결과 — 핵심 발견
1. **v1 문서와 실제 코드의 구조 불일치**
   - v1 흐름분석/총정리: 단일 Colab Worker(`garim_pipeline.py`)가 전부 처리하는 옛 구조.
   - 실제(`ocr_mask/`): **5분할 파이프라인 + `result.json` 중심**으로 재설계됨.
     - 로컬 CPU: `OCR_pipeline_report.py`, `backend_json_merger.py`, `pipeline_detail_view.py`
     - Colab: `colab_pipeline_stt.py`, `colab_pipeline_mask.py`
     - 시각(로컬 OCR) · 음성(Colab STT) **병렬 탐지**.

2. **기존 백엔드 데이터 계약 (코드 직접 확인)**
   - `detections`: 탐지 항목(label/confidence/start·end_time/bbox_x·y·w·h/detected_text).
     ⚠️ `save_pii_result()`가 **현재 음성(`voice_pii`)만 적재**, 시각 PII 경로 없음.
   - `analysis_artifacts`: 파일 산출물(`save_artifact()`, stored_path/metadata).
   - `replacement_actions.is_user_selected`: 사용자 선택 저장소(갱신 API 없음).
   - `processed_files`: 최종 결과물 + 보관기간.
   - `/analysis/jobs`(생성·조회·취소), `/worker/*`(polling/progress/results/complete) ✅.

3. **연결 공백 (❌ 미구현 API)**: 탐지목록 조회 / 상세보기 result 조회 / 선택 저장 /
   마스킹(미리보기·본처리) 트리거 / 최종 결과물 조회.

4. **프론트 현황**: `Upload→AnalysisProgress`만 실연동.
   `AnalysisReport/ReplaceOptions/Preview/Processing/Download`는 **순수 정적 mock**
   (데이터 바인딩·useState·API 없음, 레이아웃만 완성). api.js에 결과조회/선택/마스킹/다운로드 함수 없음.

## 2. ★ 연결 설계 확정 — 기존 구조에 맞춘 매핑

> 사용자 확정 원칙: 프론트/백엔드를 우리 코드에 맞추지 않고, **기존 구조 최대 재사용**.
> 비효율적이거나 더 나은 방법이 있는 부분만 신규 개발.

| 산출물 | 기존 위치 | 분류 |
|--------|----------|------|
| 음성 PII | `detections('voice_pii')` | ✅ 이미 적재 |
| 시각 PII 요약 | `detections('visual_pii'+bbox)` | 🔧 `save_pii_result()` 확장 |
| result.json·상세보기·tracks | `analysis_artifacts(stored_path)` | ✅ `save_artifact()` 재사용 |
| 사용자 선택 | `replacement_actions.is_user_selected` | 🔧 갱신 API 신규(테이블 재사용) |
| 마스킹 작업 | worker 큐(`job_type=mask_preview/mask_final`) | ✅ 큐 패턴 재사용 |
| 최종 결과물 | `processed_files` | ✅ 재사용 |

- **★ 비효율 회피**: keyframes/polygons 등 무거운 좌표는 DB 컬럼이 아닌 `result.json`
  파일(artifact)로 두고, 조회 API는 가벼운 메타만 반환. 마스킹 워커만 파일 직독 → DB 병목 회피.
- **신규 API 4종**: `GET /detections`, `GET /result`, `PUT /selections`, `GET /result-file`
  + 마스킹 Job 트리거 (모두 기존 `routes/analysis.py` 패턴 준수).

### 발견된 문제
1. **Section 4 (`audio_pii_groups`) 컬럼 오류/누락**

### 미완료 / 후속 과제
1. **Docker DB 재시작**: 기존 컨테이너는 수동 `ALTER TABLE` 실행 필요
2. **`backend_json_merger.py` artifact 등록**: merge 완료 후 `POST /worker/jobs/{job_id}/results/artifact` 호출 로직 미구현
3. **timeline_markers id 매핑 검증**: `pii_id`(timeline_markers) vs `detection_id`(detections) 불일치 가능성 → 매핑 로직 필요
4. **E2E 테스트**: Upload → AnalysisProgress → AnalysisReport → ReplaceOptions → Preview → Processing → Download 전 구간 브라우저 통과 확인
