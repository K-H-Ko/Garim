# 작업내용 요약 (2026년 06월 22일)

## 1. 개요
* **목표:** 개발(`dev`) 및 배포(`prod`)용으로 파편화되어 있던 Docker Compose 및 Dockerfile 설정들을 하나의 `docker-compose.yml` 및 범용 `Dockerfile`로 통합하여 유지보수성 향상.
* **배경:** 이미 현재 운영되는 `dev` 기반 환경이 모든 기능상 문제없이 완벽하게 동작 중이므로, 이를 기본 환경(범용)으로 삼고 사용하지 않는 파일들을 정리. 추후 도메인 연동을 위한 준비 포함.

## 2. 세부 변경 사항 (단계별 진행 내역)

### 2.1 Nginx 범용화 및 도메인 대응
* `nginx/dev.conf`를 `nginx/nginx.conf`로 이름을 변경하며, 추후 도메인 연결 시 활용할 수 있도록 `server_name` 주석 템플릿을 추가했습니다.
* Nginx 컨테이너 빌드 시 새로운 `nginx.conf`를 바라보도록 `nginx/Dockerfile`을 생성했습니다.

### 2.2 백엔드 (Backend) 범용화
* `backend/Dockerfile.dev`의 내용을 그대로 반영한 `backend/Dockerfile`을 새롭게 생성했습니다.

### 2.3 프론트엔드 (Frontend) 범용화
* `frontend/Dockerfile.dev`의 내용을 그대로 반영한 `frontend/Dockerfile`을 새롭게 생성했습니다.

### 2.4 최상위 docker-compose.yml 통합
* 단계별로 컨테이너 구동 이상 유무를 점검한 뒤, 모든 서비스가 범용 `Dockerfile`을 사용하도록 지정된 상태의 설정 내용을 최상위 `docker-compose.yml` 파일로 통합 저장했습니다.
* `docker-compose up --build` 명령어를 통해 전체 환경이 기존과 동일하고 완벽하게 작동하는 것을 검증 완료했습니다.

### 2.5 불필요 파일 정리 (Clean-up)
* 위의 검증이 모두 끝난 후, 다음 파일들을 안전하게 일괄 삭제했습니다.
  - 최상위: `docker-compose.dev.yml`, `docker-compose.prod.yml`
  - 백엔드: `backend/Dockerfile.dev`, `backend/Dockerfile.prod`
  - 프론트엔드: `frontend/Dockerfile.dev`, `frontend/Dockerfile.prod`
  - Nginx: `nginx/Dockerfile.dev`, `nginx/Dockerfile.prod`, `nginx/dev.conf`, `nginx/prod.conf`

### 2.6 garim.shop 도메인 연결 설정
* `nginx/nginx.conf` 파일에 `garim.shop` 연결 설정.
* `.env` 파일 내 `localhost` 주소들을 `garim.shop` 도메인 주소로 업데이트.
* 백엔드 CORS 도메인 허용 리스트에 `garim.shop` 추가.
* HTTP 기반으로 컨테이너 설정 후 모두 재시작.

## 3. 검증 결과 및 이슈 발생 내역
* **문제 발생:** `garim.shop`으로 설정을 마무리하고 배포하려 했으나, 사용자 환경(학원 교육실)의 제약으로 인해 공유기 접근 불가 및 **포트포워딩(80, 443 포트) 개방이 불가능**한 문제 직면.
* **상태:** DNS 설정과 Docker 컨테이너 내 Nginx, Backend, Frontend 셋팅은 완료되었으나, 외부 인터넷에서 포트포워딩 부재로 인해 로컬망을 벗어나서는 `garim.shop`으로 접근이 불가능한 상태였습니다.

## 4. 이슈 해결 (Cloudflare Zero Trust 터널 도입)
* **해결 방안 적용:** 공유기 포트포워딩 없이 외부에서 접근 가능한 **Cloudflare Tunnel**을 도입.
* **진행 내역:**
  - 도메인 네임서버를 Cloudflare로 이전하여 도메인 활성화.
  - `docker-compose.yml`에 `cloudflared` 서비스 추가 및 백그라운드 구동.
  - `.env`에 `CLOUDFLARE_TUNNEL_TOKEN` 변수 분리 및 적용.
  - Zero Trust 대시보드에서 `garim.shop` Public Hostname 라우팅 설정 완료.
  - 외부 보안 이슈 방지를 위해 `.env` 내의 모든 주소를 `http`에서 `https`로 변경.
  - Vite의 DNS Rebinding 보안 기능(`allowedHosts`)에 `garim.shop` 추가하여 차단 해제.
* **최종 상태:** 외부망(스마트폰 등)에서 `https://garim.shop` 으로 정상 접속 확인. 포트포워딩 불가능 문제 완벽 해결 및 정식 HTTPS(SSL) 적용 완료.
