## 🎯 개선1: edge 임계값 재조정 (NCC 0.70)

- edge 구간(첫/마지막 keyframe 전후)만 별도 적용
- 효과: 흐릿 등장도 감지 + 배경 표류는 여전히 방지 (0.70 이상 신뢰도)

카메라 급이동 또는 각도 급변 시 NCC 템플릿 매칭 실패

### 해결

### 적용 위치
- `_build_overlay_tracks()`: evs 생성 시 확장 박스로 NCC 템플릿 생성

### 효과

## 🎯 개선3: 추적/마스킹 박스 분리 (핵심)

### 문제 (v15 후)
v15에서 추적 확장(0.25)을 그대로 **마스킹** 범위로 사용
→ 지우는 영역이 커져 **주변 텍스트(이름 등)까지 뭉개짐**

### 해결 (v16)
**추적과 마스킹을 분리:**
- **추적용 박스**: 확장 (0.25) → NCC 템플릿 + 주변 픽셀 감지
- **마스킹 박스**: 원본 타이트 → 개인정보만 정확히 지우기

**구현:**
- `_shrink_bbox(bbox, ratio)`: 확장 박스 → 타이트 복원 (역연산)
- `_shrink_boxes(boxes, ratio)`: 확장된 모든 박스 복원
- `_vid_add_mask()`: **모든 마스킹 경로의 단일 지점**에서 추적용 → 마스킹용 축소
  ```python
  def _vid_add_mask(mask_map, fi, pii_type, boxes, bbox):
      tb = _shrink_boxes(boxes, VID_TRACK_EXPAND)  # 타이트 복원
      tbb = _shrink_bbox(bbox, VID_TRACK_EXPAND)
      mask_map.setdefault(fi, []).append((pii_type, tb, tbb))
  ```

### 환경설정
- `VID_TRACK_EXPAND = 0.25` (추적 전용 명확화)
- `VID_MASK_PAD_PX = 4` (마스킹 박스 사방 여유 px, 미세 노출만 보정)

### 검증
- 박스 왕복: 폭 175 → 확장 218.8(×1.25) → 마스킹 복원 175.0 ✓
- 효과: 추적 안정성 유지 + 주변 글자 뭉개짐 해소 ✓

## 🎯 개선4: N등분 정밀 블록 매칭 (화면밖 끝글자)

### 문제 (v16 후)
신분증이 화면 좌측으로 나가며 **끝부분이 여전히 보임**
- 추적 분리는 성공했지만, 화면밖 구간에서 **속도 외삽**만 사용 → 부정확

### 원인
- NCC 매칭 불가 (화면밖 → 템플릿 불가시)
- 마지막 속도로 추정 → 흔들림 발생

### 해결 (v17)
**텍스트를 N개 블록으로 분할 → 부분 매칭**

**환경설정:**
- `VID_BLOCK_MIN_PX = 24` (블록 최소 길이)
- `VID_BLOCK_MAX_N = 8` (최대 블록 개수)
- `VID_BLOCK_VIS_RATIO = 0.6` (화면 내 60%+ 보일 때만 신뢰)

**신규 헬퍼:**
- `_screen_vis_ratio(bbox, W, H)`: bbox의 화면 내 가시 면적 비율
- `_split_blocks(bbox)`: 긴 축 방향으로 N등분 블록 리스트

**핵심 알고리즘 (`_vid_match_shift()`):**
1. 전체 매칭 시도 (기존)
2. N등분 블록 매칭:
   - 화면 밖으로 잘린 블록(가시율 < 0.6) 제외
   - 화면 안 살아남은 블록들로만 NCC
3. 블록 2개 이상 매칭 → `estimateAffinePartial2D` (회전·스케일·이동 정밀 추정)
4. 블록 1개만 → shift (평행이동만)
5. 블록 0개 → None (완전 소멸)

**동작 예시:**
```
[1 2 3 4 5 6 7]  정상          → 7개 블록 affine 정밀
[X X 3 4 5 6 7]  좌측 화면밖   → 우측 5개 블록으로 정밀 추적 (끝글자 포함)
[X X X X X X 7]  거의 다 나감  → 끝블록 1개로 shift
[X X X X X X X]  완전 소멸     → None → 추적 종료
```

### 효과
- 화면밖 끝글자도 **실제 픽셀 이동**으로 추적
- 외삽(속도 추정)은 모든 블록 소멸 직전만 최후 수단 사용
- 결과: 자동으로 정교한 마스킹 박스 이동 ✓

### 검증
- 블록 분할 + 화면밖 시뮬레이션:
  - x=[63,238] 7개 신뢰 → x=[-45,130] 5개 → x=[-105,70] 3개 → x=[-145,30] 1개 → x=[-170,5] 0개(종료) ✓
- py_compile 통과 ✓

## 🎯 개선6: 역소급(backfill) 오매칭 차단 — 주소↔주민번호 뭉개짐 해소

### 문제 (v18 주소 패턴 추가 후 실제 영상에서 발견)
신분증 영상에서 주소와 주민번호가 서로 겹쳐 마스킹이 뭉개짐.
- 사용자 보고: "주소는 주소대로, 주민번호는 주민번호대로 각각 따로 추적·마스킹돼야 한다"

### 근본 원인 (추적/마스킹이 아니라 report 데이터 생성 단계)
`_vid_collect_pii()` 의 **역소급(backfill)** 로직 버그.
- 주소는 frame 36 부터 OCR 에 처음 잡힘 (frame 4·12·20 에는 주소 박스가 **아예 없음**)
- 역소급은 "첫 탐지 이전 프레임"에 PII 를 소급할 때 **IoU(위치 겹침)만** 사용
- → 주소가 없는 frame 4·12·20 에서 위치가 가장 가까운 **주민번호 박스**를 골라 주소를 붙임

**증거 (수정 전):** 주소_1 의 keyframe bbox 가 주민번호_1 과 frame 4·12·20 에서 완전 동일
```
frame 4 : 주민번호[324,517,482,563] == 주소[324,517,482,563]  ❌
frame 12: 주민번호[314,518,471,564] == 주소[314,518,471,564]  ❌
frame 20: 주민번호[297,519,459,566] == 주소[297,519,459,566]  ❌
frame 36: 주민번호[251,522,414,568] != 주소[253,532,467,593]  ✅(정상)
```
→ mask.py 가 주소를 추적할 때 frame 4~20 은 주민번호 자리, frame 36 은 진짜 주소 자리를
오가며 두 PII 가 겹쳐 뭉개짐.

### 해결 (역소급에 2중 가드 추가)
**환경설정:**
- `VID_BACKFILL_TEXT_SIM = 0.4` (소급 후보 박스 텍스트 최소 유사도)

**신규 헬퍼:**
- `_bbox_iou(a, b)`: 두 bbox 의 IoU
- `_overlaps_other_pii(cand, occupied, my_type)`: 후보 박스가 같은 프레임의 '다른 종류 PII'
  박스와 겹치면 True(소급 금지)

**핵심 로직 (`_vid_collect_pii` 역소급 부분):**
1. **가드①** 후보 박스가 같은 프레임의 다른 PII 점유 박스와 겹치면 제외 (주민번호 자리 회피)
2. **가드②** 후보 박스 텍스트가 원래 PII 텍스트와 `VID_BACKFILL_TEXT_SIM` 이상 유사할 때만 소급
   (주소 "시파배시미보구" vs 주민번호 "1111112111117" → 유사도 ≈ 0 → 소급 거부)

### 검증 (보존된 OCR 박스로 재현)
```
수정 후 주소 추적 frame: 36, 108 만 (진짜 주소 위치) ✅
수정 전 잘못 붙던 frame 4·12·20 주민번호 오매칭: 완전 제거 ✅
frame 36 IoU 0.39 잔여 겹침: OCR 원본의 자연스러운 줄 인접(버그 아님, 마스킹 무해)
```
- py_compile 통과 ✅

### 효과
- 주소·주민번호가 **각자 자기 위치에서 독립 추적·마스킹** → 겹침/뭉개짐 해소
- 지우는 범위도 각각 타이트하게 유지(v16 추적/마스킹 분리와 결합)
