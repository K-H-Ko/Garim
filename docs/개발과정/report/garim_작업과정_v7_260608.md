## 🎯 3단계: 화면밖 퇴장 추적 버그 수정

### 문제
신분증이 화면 좌측으로 나갈 때 추적 좌표가 흔들려 끝부분이 노출.

### 원인
NCC 매칭이 안 되는 화면밖 구간에서 **속도 외삽(extrapolation)** 사용 → 부정확

### 해결
- `_exit_extrapolate()` 함수: 마지막 매칭 위치 기반 shift 계산
- edge 전용 NCC 임계 0.70 (흐릿 등장도 감지, 표류 방지)
- backward/forward 보간 보호용으로 유지

## 🎯 추가1: 카드영상 PII 분리 + 신분증 박스 떨림

### 카드영상 특성 (카드번호_1, 카드번호_2)
- 장면 전환(컷)으로 갑자기 등장/소멸
- 연속된 움직임이 아님
- "표류" 위험: 카드 없는 구간에서 배경을 추적 시작

### 신분증 특성
- 연속 이동 후 화면밖 퇴장
- 끝부분까지 추적 필요

### 해결
- `pii_id` 단위로 **독립 추적** (카드번호_1·_2 분리)
- 각 PII의 keyframe 범위만 마스킹 → 표류 0

## 🎯 최종: 박스 표류 제거 (OCR 인식 구간만)

### 문제
카드 등장 전/소멸 후 → 배경을 PII로 잘못 추적 (표류)

### 원인
edge 외삽(`_exit_extrapolate()`)이 카드 없는 구간에서도 동작 → 보정 불가

### 해결 (사용자 결정)
- **"OCR 인식 구간만"** 원칙 실시
- `_build_overlay_tracks()`, `_vid_track_template()`:
  - 첫 keyframe '이전' backward 제거
  - 마지막 keyframe '이후' forward 제거
  - keyframe '사이'만 보간 추적

### 효과
- 카드영상: 박스 표류 0 (keyframe 범위 정확히 일치)
- 신분증: 화면밖 퇴장은 keyframe까지만 (OCR 못 읽는 구간 미탐지 trade-off)

### 최종 상태
```
카드번호_1: 박스 [0~259] = keyframe [0~259] ✓
카드번호_2: 박스 [394~475] = keyframe [394~475] ✓
등장전/소멸후/컷 표류: 완전 제거 ✓
```
