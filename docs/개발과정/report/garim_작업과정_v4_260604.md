# 작업 1 — 마스킹 방식 전환: 선형 보간 → 픽셀 템플릿 추적
**대상**: `colab_pipeline_video.py`

## 1-1. 배경
직전 선형 보간 방식의 문제: ① 12초 카드 깜빡임/노출 ② 기울어진 카드 주변 배경 뭉개짐 ③ 카드 사라진 뒤에도 마스킹 잔류.

### 사용자 핵심 결정
- **마스킹 대상은 카드번호 하나** (이름·유효기간은 개인정보 아님)
- 타이트한 범위를 **픽셀 단위로 추적** — 다음 프레임 동일 픽셀 추적, 사라지면 즉시 중지

## 1-2. 변경 내용
- `_vid_collect_pii()`: 카드번호 존 전체 확장(`mb = zone_box_list`) **삭제** → 카드번호 박스 polygon만
- `_vid_match_shift()`: `cv2.matchTemplate`(TM_CCOEFF_NORMED)로 이전 프레임 박스 픽셀 매칭, 임계 0.50 미만이면 추적 중지
- `_vid_track_run()`: 한 keyframe에서 한 방향 추적, 매칭 성공 시 polygon 평행이동
- `_vid_track_template()`: 모든 keyframe 앵커로 **양방향** 추적 (기존 `_vid_build_mask_map` 대체)
- backfill polygon 위치 보정(`_shift_boxes`)
- 파라미터: `_VID_MATCH_THRESH=0.50`, `_VID_SEARCH_RATIO=0.6`, `_VID_EDGE_MAX_SEC=2.0`

## 1-3. 검증 (카드_영상1)
- 첫 카드 0~9.98s, 둘째 카드 12.05~16.05s 연속 마스킹
- OCR 깨진 frame 369도 삭제 ✅ / 카드 진입 전 frame 360 미마스킹 ✅ / 소멸 지점 자동 종료 ✅

# 작업 2 — OCR 속도 최적화 + 세로/각도 주소 탐지 검증
**대상**: `Paddle_OCR_vidio.py`

## 2-1. 배경
픽셀 추적(±1초 커버) 도입 후 OCR을 1초 간격으로 가볍게 + 세로/각도 주소 탐지 확인 요청.

## 2-2. 로컬 실측 (paddleocr 3.6.0)
- deskew 없이 업스케일만으로 카드번호 인식 → deskew 불필요
- textline ON/OFF 탐지 동일, OFF가 21% 빠름
- 업스케일 1.3x부터 주소 정확("서울특별시")
- 한글 엔진이 카드번호 4토막을 영문 엔진과 100% 동일 인식 → 영문 2차 불필요
- 주소는 모든 설정에서 탐지됨, 정렬도 각도 기반(`assemble_text`)

## 2-3. 변경 (설정값)
| 항목 | 전 → 후 |
|------|--------|
| `SAMPLE_INTERVAL_SEC` | 0.3 → **1.0** |
| `OCR_DESKEW_ENABLED` | True → **False** |
| `OCR_EN_ENABLED` | True → **False** |
| `PADDLE_USE_TEXTLINE_ORI` | True → **False** |
| `OCR_ENHANCE_SCALE` | 1.5 → **1.3** |
| `OCR_DESKEW_DEBUG` | True → **False** |

## 2-4. 검증
- 송장(11.2초) OCR 65초 → **36.9초**, 카드(17초) **36.3초**
- 1초 간격 OCR + 픽셀추적 마스킹 구간이 0.3초 간격과 **완전 동일**

# 작업 3 — 픽셀 추적 강건화(부분 매칭) + 블러 함수 삭제
**대상**: `colab_pipeline_video.py`

## 3-1. 배경
송장 주소가 **클로즈업(줌인) 순간 다시 노출**. 카메라 급속 줌인으로 픽셀 스케일 급변 → 추적 끊김.

### 정량 진단 (frame 51 주소, bbox 96×45)
| frame | full | 좌절반 | 우절반 |
|-------|------|-------|-------|
| 51 (keyframe) | 1.00 | 1.00 | 1.00 |
| 53 | **0.48 끊김** | 0.65 | 0.86 |
| 55 | **0.44 끊김** | 0.63 | 0.74 |
| 66 | **0.46 끊김** | 0.53 | 0.68 |

→ 전체 매칭은 끊겨도 **변하지 않은 절반은 0.6~0.9 안정** → 사용자 제안("앞/뒤 일부라도") 유효 확인.

## 3-2. 변경 내용
- `_vid_ncc_probe()` 신설: NCC 측정 공통 헬퍼 `(dx,dy,score)` 반환
- `_vid_match_shift()` **부분 매칭 fallback** 추가:
  - ① 전체 매칭 ≥ 0.5 → 그대로
  - ② 실패 시 긴 축 2등분(가로→좌/우, 세로→상/하), 한쪽이라도 ≥ 0.5면 그 위치로 추적 유지
  - ③ 둘 다 미만 → 추적 중지 (임계 0.5 유지)
- 블러 함수 `_vid_mask_blur` **삭제** (복원 가능 → 보안상 금지, 호출처 없음). 모든 마스킹은 `cv2.inpaint`(Telea)만

## 3-3. 검증 (송장 영상)
| 항목 | 부분 매칭 전 | 후 |
|------|------------|----|
| frame 51~73 | 53~57·66~ 끊김(노출) | **51~73 전부 연속 마스킹** ✅ |
| 주소 구간 | 부분적 | 28~93(1.2~3.9s), 146~213(6.1~8.9s) |
| frame 72 실제 inpaint | — | 주소 삭제 / 타 텍스트·배경 보존 ✅ |

# 추가 진단 (결정 대기 — 실제 적용은 v5에서 진행 예정)

작업 3 이후 결과 영상 확인 중 발견한 2건. **진단만 완료**했고 코드 변경은 아직 안 함.

## A. frame 245(10.21초) 작은 주소 미탐지

라벨이 중간 크기로 보이는 구간에서 OCR이 주소를 못 잡음
(`[JOINED] 일산2 1577-707 321 D0- ... 37W 312E ... 23/05 W2` 수준만 인식).

### 배율별 실측 (frame 245)
| 배율 | 주소 탐지 | 속도 |
|------|----------|------|
| 1.3x (현재) | `3122(영37)` ❌ | 2.8s |
| 1.5x | `312(영용377)` ❌ | 3.9s |
| **2.0x** | **`서표제명시 영동교구 영순로10일`** ✅ (서울특별시 영등포구 영중로10길) | 5.6s |
| 2.5x | 비슷하게 ✅ | 8.0s |

→ **2.0x부터 주소 탐지됨.** 글자는 깨져도 위치가 잡히면 정규식 + 픽셀추적이 ±1초 커버.
→ 트레이드오프: 속도 2배(송장 37→약 70초). **배율 상향 여부 결정 대기.**

## B. 각도 급변 구간 마스킹 배경 침범

카메라가 빠르게 움직이며 라벨 각도가 급변하는 구간에서, 마스킹 박스가
라벨을 벗어나 옆 배경(파란 비닐)으로 일부 넘어감.

- 원인: 현재 부분 매칭이 **평행이동만** 적용 → 라벨 회전 시 polygon이 안 따라 돌아 어긋남
- 개선안: 부분 매칭의 **좌/우 두 위치로 회전·크기(affine)까지 추정**해 기울어진 라벨 추적
- 단, 회전 추정은 과거 CSRT/ORB에서 불안정했던 이력 → 구현 후 실측 검증 필수. **시도 여부 결정 대기.**

## 종합 남은 사항 / 주의

- **주소 정규식은 이미 `colab_pipeline_video.py` PII_PATTERNS(line 114)에 길게 구현됨** → 주소 마스킹 정상 동작
- 진짜 **90° 세로쓰기** 주소가 나오는 영상이면 `PADDLE_USE_TEXTLINE_ORI=True`로 되돌릴 것 (현재 송장은 ~20° 기울기라 OFF로 충분)
- 부분 매칭 임계는 0.5 유지 — 50% 미만이면 추적 중지(과잉 마스킹 방지). 너무 빨리 줌해 50% 미만으로 변형되는 극단 구간은 추적 불가(허용 한계)
- 미사용 레거시 `_vid_build_mask_map`(선형보간)은 호출되지 않음 (추후 정리 가능)
- 마스킹은 전 구간 `cv2.inpaint` Telea로만 수행 (복원 불가). 블러 함수는 삭제됨
