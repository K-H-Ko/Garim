### ② OCR 엔진 비교 및 선정 과정

영상 OCR 엔진으로 아래 4가지 후보를 검토 및 비교했습니다.

#### [후보 1] Naver CLOVA OCR (API)
- **성능**: 한글 인식 정확도 최상위 수준
- **bbox 제공**: ✅ (단어/줄 단위 정밀 좌표)
- **탈락 이유**: 프레임마다 API 호출 → 영상 1개당 수백~수천 회 호출 발생 → **비용 문제로 현재 단계 부적합**
- **향후 계획**: 상용화 단계에서 확장성 고려 시 재도입 예정

#### [후보 2] Nanonets-OCR-s (`nanonets/Nanonets-OCR-s`, Hugging Face VLM)
- **방식**: Vision-Language Model (VLM) — 이미지 전체를 보고 텍스트를 마크다운 형태로 반환
- **bbox 제공**: ❌ (전체 텍스트 문자열만 반환, 단어별 좌표 없음)
- **한글 인식**: 중간 수준 (Qwen2-VL 기반이나 한글 학습량 제한적)
- **탈락 이유**: **bbox 좌표 부재** → Phase 8 마스킹 시 정확한 PII 위치를 특정할 수 없음  
  → 마스킹 파이프라인에서 어느 픽셀 영역을 가릴지 알 수 없어 실제 활용 불가
- **현 상태**: `Nanonets_OCR_vidio.py` 파일로 코드 작성 완료 (비교 실험·인식률 테스트 목적으로 보관)

#### [후보 3] Florence-2 / Qwen2-VL (기타 VLM)
- **bbox 제공**: 제한적 (프롬프트 설정 필요, 불안정)
- **한글 인식**: Florence-2 매우 취약, Qwen2-VL 중간 수준
- **결론**: 한글 인식률과 bbox 안정성 모두 PaddleOCR에 미치지 못해 제외

#### [후보 4] PaddleOCR ✅ **최종 선정**
- **방식**: 전용 한글 OCR 모델 (`korean_PP-OCRv5`)
- **bbox 제공**: ✅ (단어/라인 단위 정밀 좌표 — 마스킹에 바로 활용 가능)
- **한글 인식**: 최상위 수준 (4개 후보 중 한글 정확도 1위)
- **오프라인 동작**: API 비용 없이 로컬 GPU/CPU에서 완전 무료 처리
- **선정 이유 요약**: 한글 인식 최강 + bbox 정확 제공 + 오프라인 무료 — 세 조건 모두 충족

### ③ `Paddle_OCR_vidio.py` 핵심 코드 설계 및 작성

PaddleOCR 기반 영상 PII 탐지 파이프라인을 구현. 주요 함수 구조:

| 함수 | 역할 |
|------|------|
| `get_paddle_engine()` | 전역 1회 모델 로드 (CUDA/CPU 자동 선택) |
| `paddle_ocr(frame)` | BGR → PaddleOCR 추론 → `(bbox, text, confidence)` 리스트 반환 |
| `dhash(image)` | 9×8 차이해시 → 64비트 정수 생성 |
| `hamming(a, b)` | 두 해시 간 비트 차이(해밍 거리) 계산 |
| `has_text_in_frame(frame)` | Canny 엣지 기반 텍스트 유무 사전 필터 (OCR 비용 절감) |
| `classify_pii(text)` | 정규식 PII 분류 → `(pii_type, text_masked)` 반환, 원본 즉시 폐기 |
| `merge_pii_hits(hits)` | 시간축 병합 — 시간 간격 ≤ 0.7초 + IoU ≥ 0.25 조건 |
| `save_visual_pii_detections()` | JSON/TXT 결과 저장 (7일 보관, 재탐지 가능 마스킹 포함) |
| `process_frame(frame, ts)` | 단일 프레임 처리 — dHash 중복 체크 → OCR → PII 분류 |
| `process_video(video_path)` | 영상 전체 처리 루프 |

**탐지 대상 PII 항목** (정규식 기반):
- 전화번호 (휴대폰 010-XXXX-XXXX, 유선 포함)
- 이름 (2~4자 한글 패턴)
- 이메일 주소
- 주민등록번호
- 주소 (시·구·동·로·길 패턴)
- 계좌번호
- 신용카드번호
- 생년월일

**재탐지 가능 마스킹 예시**:
```
010-1234-5678  →  010-****-**78   (전화번호 타입 재식별 가능)
hong@gmail.com →  ***@***.com     (이메일 타입 재식별 가능)
서울시 강남구   →  **시 **구       (주소 타입 재식별 가능)
```

### ⑤ PII 마스킹/탐지 코드 전면 제거 — 아키텍처 변경 결정

**결정 일자**: 2026년 6월 2일  
**변경 배경 및 이유**:
> JSON 출력 파일에는 OCR 결과가 단어 단위로 분리되어 저장되기 때문에,  
> 여러 단어에 걸친 PII 패턴(예: `010-1234-5678`)이 정규식으로 탐지되지 않음.  
> 또한 JSON은 DB에만 저장되어 보안 처리되며 7일 뒤 자동 삭제됨.  
> TXT·이미지 출력은 디버깅 전용으로 향후 제거 예정이라 마스킹 불필요.  
> **따라서 OCR 추출 파이프라인에서 PII 탐지·마스킹 책임을 완전히 제거하고,  
> PII 분석은 `colab_pipeline_image.py`(이미지 파이프라인)에서 전담하는 구조로 전환.**
