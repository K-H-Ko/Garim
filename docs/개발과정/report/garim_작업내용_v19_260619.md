## 📌 세션 2: 업로드 실제 파일 타입 검증 (보안 강화)

### 문제
- `media_type`(image/video)을 **프론트가 보낸 값 그대로** 저장 → 영상을 `.jpg`로 위조해 올리면 그대로 통과
- 확장자/신고값만 신뢰하는 구조 → 크레딧 적게 차감·파이프라인 오작동 위험

### 수정
| 파일 | 변경 |
|------|------|
| `backend/requirements.txt` | `filetype` 추가 |
| `backend/services/uploads.py` | `_verify_real_media_type()` 헬퍼 신규 + 병합(complete) 직후 호출 + 검증된 `media_type` 확정 저장 |

### 동작
1. 청크 병합 직후 `filetype.guess()`로 **매직 바이트** 판별 (확장자 무시)
2. 허용 목록 검사 — 이미지 `jpg/png/webp`, 영상 `mp4/avi/mov/mkv` 외 → 거부
3. 신고값(image/video) ≠ 실제 대분류 → 거부 (위조 차단)
4. 거부 시: 병합 파일 삭제 + 상태 `failed` + 실제 형식 안내 메시지 (`ValueError` → 400, Upload.jsx가 그대로 표시)
5. 통과 시 검증된 실제 타입을 DB `media_type`에 확정 저장

> ⚠️ 운영 반영 시 `pip install filetype` (또는 컨테이너 재빌드) 필요

## 📌 세션 3: 전역 라이트/다크 테마 인프라 구축

### 방향
- 라이트(기본) = 현재 화이트, 최초 진입은 항상 라이트
- 다크 = 가독성 우선 **중성 차콜 톤** (요금제 페이지만 추후 골드 액센트 예정)
- 방식: `<html data-theme="dark">` 속성 토글 → CSS 변수 오버라이드

### 작업
| 파일 | 변경 |
|------|------|
| `context/ThemeContext.jsx` (신규) | `useTheme()` 훅 — localStorage 저장/복원, `<html data-theme>` 적용 |
| `App.jsx` | `<ThemeProvider>`로 전체 래핑 |
| `GarimHeader.jsx` | 헤더 우측에 해/달 **테마 토글 버튼** 추가 (public·app·landing·admin 공통) |

### 변수화한 하드코딩 색상 (라이트 동작 유지, 다크 자동 대응)
- `.gh`(헤더) `#fff` → `var(--header-bg, #fff)`
- `.gf`(풋터) `#fff` → `var(--footer-bg, #fff)`
- `.page-public` `#fff` → `var(--page-public-bg, #fff)`
- `.section--alt` `#fafafa` → `var(--surface-alt)`
- `.stat-card`·`.feature-card`·`.price-card`·`.problem-card` `#fff` → `var(--bg-1)`
- `.hero` 흰색 그라데이션 → 다크 전용 그라데이션 오버라이드

> 참고: 풋터/헤더 로고 invert는 추가했다가 **로고 색상 유지 위해 사용자가 비활성화(주석 처리)**
